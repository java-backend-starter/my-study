# OSI 7계층 모델 상세 요약

OSI 7계층(Open Systems Interconnection Model)은 네트워크 통신 과정을 7개의 계층으로 나누어 정의한 국제 표준 모델입니다. 각 계층은 독립적으로 작동하며 상·하위 계층과 데이터를 주고받으면서 전체 통신 과정을 구성합니다.

---

## 1. 물리 계층 (Physical Layer)
- 역할: 데이터를 비트(bit) 단위로 전기적/광학적 신호로 변환하여 실제 물리 매체(케이블 등)를 통해 전송
- 기능:
    - 하드웨어적 연결 수립 및 해제
    - 전송 매체에 따라 전기/광학 신호 처리
    - 전송 속도(baud rate), 전압 수준, 커넥터 종류 등 정의
- 장비: 허브, 리피터, 케이블, 모뎀, NIC(Network Interface Card)
- 데이터 단위: 비트(Bit)

---

## 2. 데이터 링크 계층 (Data Link Layer)
- 역할: 물리 계층을 통해 수신된 데이터를 프레임 단위로 구성하여 신뢰성 있는 전송 제공 (노드 간 통신)
- 기능:
    - MAC 주소 기반 데이터 전송
    - 프레임 단위의 데이터 전송
    - 오류 검출(FCS), 흐름 제어, 재전송
- 프로토콜: Ethernet, PPP, HDLC, ARP
- 장비: 스위치, 브리지
- 데이터 단위: 프레임(Frame)

---

## 3. 네트워크 계층 (Network Layer)
- 역할: 패킷을 목적지까지 전달하기 위한 논리적 주소 지정(IP)과 라우팅 수행
- 기능:
    - 경로 선택(Routing)
    - IP 주소 지정
    - 패킷 분할 및 재조립
    - 혼잡 제어
- 프로토콜: IP, ICMP, ARP, RIP, OSPF
- 장비: 라우터, L3 스위치, 방화벽
- 데이터 단위: 패킷(Packet)

---

## 4. 전송 계층 (Transport Layer)
- 역할: 송신자와 수신자 간의 종단 간(end-to-end) 신뢰성 있는 데이터 전송 보장
- 기능:
    - 연결 설정 및 해제
    - 흐름 제어, 오류 검출, 재전송
    - 데이터 분할 및 순서 제어
- 프로토콜:
    - TCP: 연결 지향, 신뢰성 보장
    - UDP: 비연결성, 빠르지만 비신뢰성
- 데이터 단위: 세그먼트(TCP) / 데이터그램(UDP)

---

## 5. 세션 계층 (Session Layer)
- 역할: 통신 세션을 관리 (설정, 유지, 종료)
- 기능:
    - 세션 간 동기화 및 회복
    - 대화 제어 (full-duplex, half-duplex)
- 예시: 원격 접속, 화상 회의, RPC
- 비고: 오늘날에는 전송 계층이나 응용 계층에서 기능 대체

---

## 6. 표현 계층 (Presentation Layer)
- 역할: 데이터의 형식 및 구문을 맞춰 시스템 간 호환성 유지
- 기능:
    - 인코딩/디코딩
    - 데이터 암호화/복호화
    - 압축/해제
- 예시: JPEG, MPEG, ASCII, EBCDIC, SSL/TLS
- 비고: 응용 계층에 통합되는 경우 많음

---

## 7. 응용 계층 (Application Layer)
- 역할: 사용자가 직접 접하는 계층으로, 응용 프로그램이 네트워크 서비스를 사용할 수 있도록 인터페이스 제공
- 기능:
    - 사용자 인증
    - 파일 전송, 이메일, 웹 브라우징 등
- 프로토콜: HTTP, FTP, SMTP, POP3, IMAP, DNS
- 예시: 크롬, 지메일, 파일질라, Outlook
- 데이터 단위: 데이터(Data)

---

## 요약표

| 계층 번호 | 계층 이름        | 주요 역할 및 장비/프로토콜         |
|-----------|------------------|------------------------------------|
| 7         | 응용 계층         | 사용자 인터페이스, HTTP/FTP       |
| 6         | 표현 계층         | 암호화, 인코딩, 데이터 형식 변환   |
| 5         | 세션 계층         | 세션 관리, 대화 제어               |
| 4         | 전송 계층         | TCP/UDP, 오류 복구                 |
| 3         | 네트워크 계층     | IP 주소, 라우팅, 패킷 전달         |
| 2         | 데이터 링크 계층  | MAC 주소, 스위치, 오류 제어        |
| 1         | 물리 계층         | 비트 전송, 케이블, 허브            |
