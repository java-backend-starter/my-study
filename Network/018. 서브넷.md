# 서브넷

---

## 1.서브넷(Subnet)

- 하나의 네트워크를 더 작은 단위로 나누어 관리하는 방법 
- 이러한 서브넷을 만들 때 사용하는 것이 서브넷 마스크(Subnet Mask)로, IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 역할
- 서브넷팅(Subnetting)은 네트워크를 여러 개의 서브넷으로 분할하는 과정을 의미하며, 이를 통해 IP 주소의 효율적인 사용과 네트워크 성능 향상을 도모할 수 있음

---

## 2. 서브넷과 서브넷 마스크

- 서브넷은 IP 주소에서 네트워크 영역을 부분적으로 나눈 부분망을 의미 
- 서브넷 마스크는 이러한 서브넷을 만들기 위해 사용되는 32비트의 이진수로, 네트워크 ID와 호스트 ID를 분리하는 역할
- 예를 들어, C클래스 IP 주소인 192.168.32.0의 기본 서브넷 마스크는 255.255.255.0이며, 이를 통해 네트워크 ID와 호스트 ID를 구분할 수 있습니다.

---

## 3. 서브넷팅의 필요성

1. IP 주소의 효율적 사용:
   - 서브넷팅을 통해 IP 주소를 적절하게 분배하여 낭비를 방지할 수 있습니다. 
   - 예를 들어, 특정 부서에 30개의 IP 주소만 필요하다면, 
   - 전체 네트워크를 분할하여 필요한 만큼만 할당함으로써 주소 낭비를 줄일 수 있음
2. 네트워크 성능 향상:
   - 서브넷팅은 브로드캐스트 도메인을 나누어 브로드캐스트로 인한 성능 저하를 막는 데 도움이 됨
   - 서브넷 간의 통신은 라우터가 담당하며, 이를 통해 네트워크의 효율성을 높일 수 있음

---

## 4. 서브넷팅의 예시

- C클래스 IP 주소인 192.168.32.0/24를 서브넷팅하여 서브넷 마스크의 비트 수를 1 증가시켜 192.168.32.0/25로 변경하면, 네트워크를 두 개로 분할할 수 있음 
- 이렇게 하면 각 서브넷은 126개의 호스트를 가질 수 있으며, 네트워크 주소와 브로드캐스트 주소를 제외한 나머지 주소를 호스트에 할당할 수 있음
- 서브넷팅을 통해 네트워크를 효율적으로 관리하고, IP 주소의 낭비를 방지하며, 네트워크 성능을 향상시킬 수 있음

---

## 5. 슈퍼넷팅(Supernetting)

- 인터넷이 매년 폭발적으로 성장하면서, 1992년경 IETF에서는 인터넷 라우팅 시스템의 확장성에 대한 심각한 우려를 제기
- 주요 문제
  - Class A와 Class B 네트워크 주소 공간의 고갈
  - Class C 주소가 대량 할당되면서 라우팅 테이블의 크기 증가
  - 32비트 IPv4 주소 공간의 궁극적인 고갈
- 이러한 문제를 해결하기 위해 IETF는 '슈퍼넷팅(Supernetting)' 혹은 'CIDR (Classless Inter-Domain Routing)'이라는 새로운 방식을 도입했고, 1993년 9월에 이를 표준화
- 슈퍼넷팅은 기존의 클래스 기반 주소 체계에서 벗어나, 네트워크 주소를 보다 유연하게 할당하고 묶어서 사용하는 방식
- 주요 효과
  - 라우팅 테이블의 크기 감소
  - IP 주소 공간의 효율적인 사용
  - 주소 고갈 문제의 완화
- 궁극적인 해결책은 128비트 주소 체계를 사용하는 IPv6로 전환하는 것 
- IPv6로의 전환에는 기술적/운영적 문제들이 선행되어야 하므로, 과도기적으로 IPv4 환경에서 슈퍼넷팅이 사용
