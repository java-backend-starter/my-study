# DHCP(Dynamic Host Configuration Protocol)

---

## 1. DHCP란?

- DHCP(Dynamic Host Configuration Protocol)는 네트워크에 연결되는 디바이스에게 IP 주소와 관련 정보를 자동으로 할당해주는 프로토콜이다. 
- 수동 설정 없이 IP 주소 할당을 자동화하여 네트워크 관리의 효율성을 높여준다.

---

## 2. 작동 원리 (클라이언트-서버 모델)

- DHCP는 4단계를 거쳐 작동한다:
  - DHCP Discover 
    - 클라이언트가 DHCP 서버를 찾기 위해 브로드캐스트 메시지를 전송한다. 
  - DHCP Offer
    - 서버가 사용 가능한 IP 주소와 설정 정보를 담은 메시지를 클라이언트에 응답한다. 
  - DHCP Request
    - 클라이언트가 원하는 IP 주소를 요청하는 메시지를 서버에 전송한다. 
  - DHCP Acknowledge 
    - 서버가 IP 주소 할당을 확정하고, 클라이언트는 해당 주소를 사용하여 네트워크 통신을 시작한다.

---

## 3. IP 주소 임대(Lease)

1. DHCP 서버는 임시로 IP 주소를 할당한다. 
2. 임대 기간이 만료되면, 클라이언트는 IP 주소를 갱신하거나 새로 요청한다. 
3. 이 방식은 IP 충돌 방지 및 주소 재활용에 효과적이다.

---

## 4. DHCP 서버 설정 및 관리

- IP 주소 풀 설정: 할당 가능한 IP 범위를 지정 
- 제외된 주소 설정: 고정적으로 사용해야 할 주소는 제외 
- 기타 설정: 서브넷 마스크, 게이트웨이, DNS 서버 등 설정 가능 
- 임대 기간 설정: 임대 주기를 설정하여 네트워크 상황에 맞게 조정


---

## 5. 장점

- IP 주소 자동 할당 
- 사용하지 않는 IP 주소 재활용 
- IP 충돌 최소화 
- 중앙 집중식 관리 가능

---

## 6. 단점

- DHCP 서버가 다운되면 새 디바이스는 IP 주소를 받을 수 없음 
- 보안 취약성 존재 (악성 서버의 응답 가능성)
- 일부 환경에서는 정적 IP가 필요해 DHCP가 적합하지 않음
