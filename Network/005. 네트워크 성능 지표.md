# 네트워크 성능 지표

---

## 0. 개요

- 네트워크 성능을 평가하는 데 중요한 네 가지 지표는 대역폭(Bandwidth), 대기 시간(Latency), 지터(Jitter), 패킷 손실(Packet Loss)

---

## 1. 대역폭 (Bandwidth)

- 네트워크가 단위 시간에 전송할 수 있는 최대 데이터 양을 의미함.
- 일반적으로 bps(초당 비트 수)로 측정됨.
- 비유: 고속도로의 차선 수에 비유할 수 있으며, 차선이 많을수록 더 많은 차량이 동시에 이동 가능.

---

## 2. 대기 시간 (Latency)

- 데이터 패킷이 한 지점에서 다른 지점으로 이동하는 데 걸리는 시간.
- 밀리초(ms) 단위로 측정됨.
- 구성 요소:
    - 처리 지연
      - 데이터 처리 시간
      - 캡슐화 해제, 헤더 분석, 데이터 추출, 오류 확인, 경로 선택 등의 작업이 포함 
      - 일반적으로 고속 라우터의 처리 지연은 마이크로초 이하
    - 대기열 지연
      - 네트워크 장비의 큐에서 대기하는 시간
      - 현재 대기열에 전송되고 있는 다른 패킷이 있는지 여부에 따라 달라짐 
      - 대기열이 비어 있고 전송 중인 다른 패킷이 없는 경우 해당 패킷의 대기열 지연은 XNUMX
      - 트래픽이 많고 다른 많은 패킷도 전송을 기다리고 있는 경우 대기열 지연이 상당히 클 수 있음
      - 일반적으로 밀리초에서 마이크로초 범위
    - 전송 지연
      - 라우터와 스위치가 데이터를 전송하는데 걸리는 시간
      - 라우터의 대기열이 패킷을 네트워크 링크로 전달하는 데 필요한 시간
      - 실제 전송 지연은 일반적으로 밀리초에서 마이크로초 범위
    - 전파 지연
      - 메시지가 두 라우터 간의 물리적 링크를 통해 이동하는 데 걸리는 시간
      - 두 라우터 사이의 거리를 링크의 전파 속도로 나눈 값과 동일하며, (D/S)로 표시 
      - (D)는 두 라우터 사이의 거리, (S)는 다음의 전파 속도
      - 실제 전파 지연은 밀리초 단
- 측정 방법: `ping` 명령어로 측정 가능.

---

## 3. 지터 (Jitter)

- 연속적인 데이터 패킷 간의 도착 시간 변동.
- 원인
    - 네트워크 혼잡
    - 타이밍 드리프트
    - 경로 변경
- 영향: 온라인 게임, 화상 회의 등 실시간 데이터 전송에 치명적.
-  예시 : 웹 사이트에 액세스할 때 발생하는 최대 지연 시간이 10ms이고 최소 지연 시간이 5ms인 경우 네트워크 지터는 5ms

---

## 4. 패킷 손실 (Packet Loss)

- 데이터 패킷이 목적지에 도달하지 못하는 현상.
- 수신 측에서 누락된 데이터를 감지하면 시퀀스 번호를 기반으로 손실된 패킷의 재전송을 요청
- 주 원인: 네트워크 혼잡
  - . 네트워크 장비가 처리하기에는 데이터 트래픽이 너무 많으면 일부 패킷이 필연적으로 손실될 수 있음
- 패킷 손실률: (손실된 패킷 수 / 총 전송된 패킷 수) × 100%
- 영향: 높은 패킷 손실률 → 네트워크 성능 저하 및 사용자 경험 악화.
