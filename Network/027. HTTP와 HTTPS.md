# HTTP와 HTTPS

---

## 1. HTTP란?

- HTTP(HyperText Transfer Protocol)는 클라이언트와 서버 간 데이터를 주고받기 위한 프로토콜
- 웹에서 하이퍼텍스트 전송에 사용됨
- 80번 포트를 사용
- TCP/IP 위에서 작동하는 애플리케이션 레벨의 프로토콜
- Stateless(상태 비유지)
- 암호화되지 않은 평문 데이터 전송 → 보안에 취약

### HTTP 메시지 구성

1. Method
    - 요청의 동작을 나타냄 (예: GET, POST, PUT, DELETE 등)
2. Path
    - 요청할 리소스의 경로 (예: /login, /api/user 등)
3. Version
    - HTTP 프로토콜의 버전 (예: HTTP/1.1, HTTP/2 등)
4. Headers
    - 요청/응답에 대한 메타정보를 포함 (예: Content-Type, User-Agent 등)
5. Body
    - 전송할 실제 데이터 (POST, PUT 등의 요청에서 주로 사용)

## 2. HTTPS란?

- HTTPS(HyperText Transfer Protocol Secure)는 HTTP + 보안(SSL/TLS)
- 443번 포트를 사용
- 데이터 암호화를 통해 네트워크 상의 도청/변조 방지

### 암호화 방식

- HTTPS는 대칭키 암호화 방식과 비대칭키 암호화 방식을 모두 사용한다.


#### 대칭키 암호화 (Symmetric Encryption)

- 클라이언트와 서버가 동일한 키를 사용해 데이터를 암호화/복호화
- 장점: 연산 속도가 빠름
- 단점: 키가 노출되면 모든 데이터가 위험함

#### 비대칭키 암호화 (Asymmetric Encryption)

- 공개키(Public Key)와 개인키(Private Key) 한 쌍으로 구성
- 서로 짝을 이루며, 한 키로 암호화한 데이터는 다른 키로만 복호화 가능
- 장점: 키가 공개되어도 비교적 안전함
- 단점: 연산 속도가 느림

#### 공개키(Public Key)

- 누구나 사용할 수 있도록 공개된 키

#### 개인키(Private Key)

- 오직 나만 알고 있어야 하는 비밀 키

#### 암호화 방식에 따른 효과

| 암호화 방식       | 복호화 키   | 효과 |
|------------------|-------------|------|
| 공개키 암호화     | 개인키      | 오직 키 소유자만 데이터 열람 가능 → 기밀성 보장 |
| 개인키 암호화     | 공개키      | 모두가 확인 가능 → 정보의 출처를 인증, 신뢰성 보장 |

- 암호화에 어떤 키를 사용하느냐에 따라 보안 목적이 달라진다.

### HTTPS 동작 과정

1. 클라이언트가 서버에 연결 시도
2. 서버가 인증서(공개키 포함)를 클라이언트에 전송
3. 클라이언트가 인증서를 검증
4. 클라이언트가 세션키를 생성하고 서버의 공개키로 암호화하여 전송
5. 서버가 개인키로 세션키 복호화
6. 이후 세션키로 양방향 데이터 암호화 통신 수행

### 인증서 발급 과정

1. 서버: 공개키/개인키 생성
2. 서버 → CA(인증기관)에게 인증서 발급 요청
3. CA는 서버 정보를 담아 인증서를 발급하고 서명함
4. 클라이언트는 CA의 공개키로 인증서 유효성 검증 및 서버 공개키 획득

## 3. HTTP vs HTTPS

| 항목        | HTTP                          | HTTPS                                 |
|-------------|-------------------------------|----------------------------------------|
| 보안성       | 낮음 (평문 통신)                | 높음 (암호화 통신)                     |
| 속도         | 빠름                            | 느림 (암호화/복호화 과정)              |
| 포트 번호    | 80번                            | 443번                                   |
| 인증서 필요 | 불필요                          | 필요 (CA를 통한 SSL 인증서)            |
| 사용 용도    | 비민감 데이터 전송               | 민감 정보(로그인, 결제 등) 전송         |

---

## 사용 권장 사항

- 로그인, 결제, 개인정보 등 민감 정보 전송 시 반드시 HTTPS 사용
- 단순 정보 조회 등에는 HTTP 사용 가능하지만 HTTPS 권장
