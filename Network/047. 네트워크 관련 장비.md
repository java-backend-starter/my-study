# 네트워크 관련 장비

---

## 1. 네트워크 인터페이스 카드 (NIC: Network Interface Card)

- 정의: 컴퓨터가 네트워크에 접속할 수 있게 해주는 장치.
- 기능:
  - 컴퓨터와 네트워크 간의 물리적 연결 제공 
  - 송수신 데이터의 전기적 신호 변환
- 별칭: 이더넷 카드, LAN 카드, 네트워크 어댑터
- 형태: 유선/무선 NIC, PCI 슬롯 삽입형 또는 메인보드 내장형 등 다양
- 예시: 데스크탑에 장착된 무선 랜카드

---

## 2. 허브 (Hub)
   
- 정의: 근거리 컴퓨터들을 물리적으로 연결하는 장치 
- 기능
  - 각 회선(컴퓨터 연결)을 통합적으로 연결 
  - 신호 증폭(리피터 역할 포함)
- 종류 
  - 더미 허브 (Dummy Hub)
    - 단순한 데이터 전송만 수행 (브로드캐스트 방식)
    - 대역폭을 모든 연결된 장치가 공유 
    - 각 노드를 물리적 성형 구조로 연결
    - 예시: 구형 소규모 사무실 네트워크
  - 스위칭 허브 (Switching Hub)
    - 포트 간 직접 연결로 충돌 도메인 분리 
    - 각각의 노드에 전용 대역폭 제공 
    - 패킷을 목적지 MAC 주소 기반으로 전송 
    - 예시 : 현재 사용되는 허브 대부분은 스위칭 허브

---

## 3. 리피터 (Repeater)
   
- 정의: 신호가 약해지거나 왜곡됐을 때 원래 상태로 재생하여 전송 
- 동작 계층: OSI 1계층(물리 계층)
- 기능
  - 전송 거리 연장 
  - 배선 구조 유연성 제공
  - 근거리 네트워크 간 연결
- 예시: 와이파이 중계기, 광케이블 증폭기
  - 
---

## 4. 브리지 (Bridge)
   
- 정의: LAN과 LAN 또는 LAN 내 세그먼트를 연결하는 장치 
- 동작 계층: OSI 2계층(데이터 링크 계층), 특히 MAC 계층 
- 기능
  - MAC 주소 기반 필터링 및 포워딩 
  - 트래픽 병목 완화 
  - 보안성과 효율 향상
- 기타
  - 브리지를 n개 사용하면 연결 가능한 회선 수 = n(n-1)/2 
  - 서브넷 구성에 활용
- 예시: 두 개의 이더넷 LAN을 연결하는 장치
  
---

## 5. 스위치 (Switch)
   
- 정의: 브리지와 유사하지만 더 빠르고 다기능인 장치 
- 동작 계층: OSI 2계층(데이터 링크 계층)
- 기능
  - 하드웨어 기반으로 프레임 처리 → 빠른 전송 
  - 포트별로 독립적인 전송 지원 
  - 수십~수백 개의 포트 제공 가능
- 비고: 일부 고급 스위치는 3계층(라우팅 기능)까지 지원
- 예시: 기업용 네트워크에서 스위치로 수십 대 PC 연결

---

## 6. 라우터 (Router)
   
- 정의: 서로 다른 네트워크 간 데이터를 연결하고 최적 경로로 전송 
- 동작 계층: OSI 3계층(네트워크 계층)
- 기능
  - 라우팅 테이블 기반 경로 선택 
  - LAN 간, LAN-WAN 간 연결 
  - 프로토콜 구조가 다른 네트워크 간 프로토콜 변환
- 예시: 가정용 인터넷 공유기, 기업 라우터

---

### 7. 게이트웨이 (Gateway)
   
- 정의: 프로토콜 구조가 다른 네트워크 전체(1~7계층)를 연결 
- 동작 계층: 전 계층(1~7계층)
- 기능
  - 데이터 형식, 주소, 프로토콜 변환 
  - 응용 계층 포함 고수준 연결 가능 
  - 다른 네트워크로의 출입구 역할
- 예시: 메일 게이트웨이, VoIP 게이트웨이

| NIC         | -                | 컴퓨터와 네트워크 간 물리적 연결 제공        | X            | 유선/무선 랜카드             |

| 허브        | 물리 계층 (1계층) | 단순 브로드캐스트 방식, 모든 포트에 전송     | X            | 구형 허브                    |

| 브리지      | 데이터 링크 (2계층) | LAN과 LAN 연결, MAC 주소 필터링               | O            | 두 개의 LAN 연결             |
| 스위치      | 데이터 링크 (2계층, 일부 3계층 지원) | 빠른 프레임 처리, 다수 포트 제공     | O            | 기업용 네트워크 스위치       |
| 라우터      | 네트워크 계층 (3계층) | 라우팅, 최적 경로 선택, 서로 다른 네트워크 연결 | O         | 공유기, 인터넷 라우터        |
| 게이트웨이  | 전 계층 (1~7계층) | 프로토콜/주소/데이터 형식 변환, 고수준 연결   | O            | 메일 게이트웨이, VoIP 장비    |

## 9. 네트워크 장비 설치 시 고려 사항
   
- 장비 활용 극대화 
  - 자원 낭비를 줄이고 최대 효율로 구성
- 확장성 고려 
  - 향후 시스템 증설, 기술 도입 대응
- 최신 버전 선택 
  - 호환성과 안정성을 위한 최신 하드웨어/소프트웨어 사용
- 트래픽 분산 설계 
  - 네트워크 병목 방지
- 유지보수 용이성 
  - 관리가 쉽고 신속한 점검 가능 구조
- 장애 대응력 확보 
  - 여유 포트를 통한 긴급 대처 가능
- 미래 지향성 확보 
  - 신기술 연동 가능성 고려한 설계
