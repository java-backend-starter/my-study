# 회사에서 특정 사이트 차단하는 방법과 우회하는 방법

---

## 1. 회사에서 특정 사이트 차단하는 방법

### 방화벽(Firewall) 설정

- 네트워크의 입출입 트래픽을 통제하여 특정 웹사이트 접근을 차단한다.
- 차단 방법
    - 도메인(URL) 또는 IP 주소 기준으로 트래픽 차단 룰 설정.
    - 포트 번호(TCP 80, 443 등) 제한 가능.
- 예시 제품
    - FortiGate, Palo Alto, SonicWall 등.

### DNS 차단

- 사용자의 도메인 이름 요청을 차단하거나 엉뚱한 IP로 응답하여 접속 불가하게 한다.
- 차단 방법
    - 내부 DNS 서버에서 특정 도메인 이름 해석 불가 설정.
    - 외부 필터링 DNS(예: KT DNS Clean) 서비스 사용.
- 주의
    - DNS over HTTPS(DoH) 사용 시 우회 가능성 존재.

### 프록시 서버 설정

- 네트워크 트래픽을 프록시 서버를 거치게 하여 허용된 사이트만 접속하게 한다.
- 차단 방법
    - 스쿼드(Squid) 같은 오픈소스 프록시 서버 설치 및 화이트리스트 설정.
    - 인증 프록시를 통해 사용자별 접근 제어.
- 추가 기능
    - 접근 기록 로깅 및 감사 기능.

### 웹 필터링 솔루션 사용

- 전문 솔루션을 통해 사이트 카테고리별로 접속을 차단합니다.
- 차단 방법
    - "게임", "SNS", "음란물" 등 카테고리 별 정책 적용.
    - 실시간 사이트 분류 및 차단 가능.
- 예시 제품
    - 시큐어웹, 블루코트(Blue Coat), 시만텍 웹필터 등.

### 라우터 차단 (ACL 설정)

- 네트워크 라우터에 ACL(Access Control List)을 설정해 차단합니다.
- 차단 방법
    - 특정 IP 또는 포트로 향하는 트래픽을 직접 거부(Deny) 설정.

## 2. 회사 차단 우회 방법

- 회사 보안 시스템은 매년 진화하고 있으며, SSL 검사를 통해 VPN/프록시/DoH 트래픽까지 차단하는 경우가 있다.
- 우회 시도는 내부 보안 정책 위반으로 간주될 수 있으므로 반드시 주의해야 한다.

### **아래 내용은 면접 준비를 위해 학습한 것이며, 실제 사용 의도는 없고 회사 보안 정책을 철저히 준수할 예정입니다.**


### VPN 사용

- 가상 사설망(VPN)으로 모든 트래픽을 암호화하여 외부 서버를 통해 우회한다.
- 대표 서비스
    - NordVPN, ProtonVPN, ExpressVPN, Outline VPN.
- 특징
    - IP 주소 숨김, 트래픽 암호화.
    - 일부 기업은 VPN 트래픽 자체를 탐지하여 차단할 수 있음.

### 프록시 서버 이용

- 공용 웹 프록시 사이트를 통해 차단된 웹사이트에 접속한다.
- 대표 프록시
    - HideMyAss, KProxy, CroxyProxy.
- 주의
    - 보안이 취약할 수 있어 개인정보 노출 위험 존재.

### DNS 변경

- 회사 DNS 대신 공용 DNS 서버를 사용해 차단 우회 시도.
- 대표 DNS
    - 구글 DNS: 8.8.8.8 / 8.8.4.4
    - 클라우드플레어 DNS: 1.1.1.1
- 특징
    - DNS 차단만 된 경우 유효.
    - 방화벽 차단에는 효과 없음.

### Tor 브라우저 사용

- Tor 네트워크를 통해 여러 노드를 경유하여 익명으로 사이트 접속.
- 특징
    - 트래픽 익명화 및 암호화.
    - 접속 속도가 느릴 수 있음.
    - 일부 회사에서는 Tor 트래픽 탐지 및 차단 가능.

### SSH 터널링

- 외부 SSH 서버에 접속하여 포트 포워딩으로 트래픽을 전달.
- 설정 방법
    - `ssh -D 1080 user@remote-server` 명령으로 SOCKS 프록시 생성.
    - 로컬 브라우저에 SOCKS 프록시 설정.
