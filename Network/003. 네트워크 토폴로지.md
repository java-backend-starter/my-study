# 네트워크 토플로지

---

## 1. 네트워크 토폴로지(Network Topology)

* 네트워크 토폴로지는 네트워크를 구성하는 요소들(링크, 노드)의 배치 형태를 의미
* 주요 방식으로 Star, Bus, Ring, Tree, Mesh 방식이 존재

### Star 방식

- 특징: 중앙 노드에 각 단말기가 직접 연결되는 방사형 구조
- 장점
  - 설치와 유지보수가 용이함
  - 고장 발생 시 문제 지점 파악이 쉬움
  - 각 노드별 전송 속도 설정이 가능함
- 단점
  - 많은 통신 회선이 필요함
  - 중앙 노드에 장애가 발생하면 전체 네트워크에 영향을 미침

### Bus 방식

- 특징: 하나의 통신 회선에 여러 노드가 연결되며, 브로드캐스팅 방식으로 데이터를 전송함
- 장점
  - 통신 회선을 공유하므로 경제적임
  - 단말기 증설이나 축소가 용이함
- 단점
  - 연결된 단말기 수가 증가하면 전송 속도가 저하됨
  - 모든 단말기가 데이터를 수신하므로 보안에 취약할 수 있음

### Ring 방식

- 특징: 각 노드가 양 옆의 두 노드와 연결되어 단방향으로 데이터가 전송되는 구조임
- 장점
  - FDDI 방식에서는 한쪽 회선에 장애가 발생해도 다른 방향으로 통신이 가능하여 신뢰성이 높음
  - 노드 연결이 최소화되어 고장 지점 파악이 용이함
- 단점
  - 노드 증설이나 축소가 어려움
  - 모든 노드가 중계 기능을 수행해야 하므로 전송 속도가 지연될 수 있음

### Tree 방식

- 특징: 중앙 컴퓨터를 최상위로 하여 계층적으로 하위 노드들이 연결되는 구조
- 장점
  - 통신망의 확장과 유지 관리가 용이함
  - 네트워크를 분할하여 체계적인 구성이 가능함
- 단점
  - 상위 노드에 장애가 발생하면 하위 노드의 통신에 영향을 미침

### Mesh 방식

- 특징: 모든 노드가 서로 직접 연결된 그물형 구조
- 장점
  - 어떤 회선에 장애가 발생해도 다른 경로를 통해 통신이 가능하여 신뢰성이 높음
- 단점
  - 각 노드마다 모든 다른 노드와의 연결이 필요하므로 비용이 많이 듦
  - 중요한 네트워크가 아니면 잘 사용되지 않음
