# CIDR

---

## 1. 개념

- IP 주소를 효율적으로 할당하고 라우팅을 최적화하기 위한 주소 표기법
- IP 낭비를 줄이고 라우팅 테이블 크기를 줄이는 데 목적이 있음

---

## 2. CIDR 표기법

- 형식: IP주소/비트수
- 슬래시(/) 뒤에 나오는 숫자는 네트워크 부분의 비트 수를 나타냄

### 예시

- IPv4
  - 10.0.0.0/8    -> 서브넷 마스크: 255.0.0.0   -> 범위: 10.0.0.0 ~ 10.255.255.255
  - 192.168.1.0/24 -> 서브넷 마스크: 255.255.255.0 -> 범위: 192.168.1.0 ~ 192.168.1.255
- IPv6
  - 2001:0db8::/32 -> 범위: 32비트 고정된 네트워크
  - fe80::/64      -> 링크 로컬 주소로 자주 사용

---

## 3. 장점

- 클래스 기반 방식보다 유연함 (A, B, C 클래스 구분 X)
- 필요한 IP 수에 맞게 세분화 가능
- 라우팅 테이블 크기 감소 (라우트 집약 가능)

---

## 4. 서브넷 마스크와의 관계

- /24 => 255.255.255.0 -> 256개 주소
- /30 => 255.255.255.252 -> 4개 주소 (소규모 네트워크에 사용)

---

## 5. 네트워크 결합 예시 (Supernetting)

- 192.168.1.0/24 + 192.168.2.0/24
  => 192.168.1.0/23 으로 통합 가능

---

## 6. 서브넷팅 (Subnetting)

- 큰 네트워크를 작은 서브넷으로 나누기
- 필요한 수의 IP만 할당 가능
- 자세한 내용은 [서브넷](https://github.com/seonghwanJang/my-study/blob/main/Network/018.%20%EC%84%9C%EB%B8%8C%EB%84%B7.md)

---

## 7. 슈퍼네팅(Supernetting)

- 여러 개의 연속된 IP 네트워크를 하나의 큰 네트워크로 결합하는 기술
- 라우팅 테이블의 크기 감소 및 네트워크 효율성 향상 목적

### 슈퍼네팅의 장점

- 라우팅 테이블 축소 → 라우터 성능 향상
- IP 주소 공간 절약 → 주소 낭비 방지
- 대규모 네트워크 관리 용이

### 서브네팅 vs 슈퍼네팅

- 서브네팅: 하나의 큰 네트워크 → 여러 개의 작은 네트워크
- 슈퍼네팅: 여러 개의 작은 네트워크 → 하나의 큰 네트워크

### 예시

- 네 개의 /24 네트워크 
  - 192.168.0.0/24
  - 192.168.1.0/24 
  - 192.168.2.0/24
  - 192.168.3.0/24
- 슈퍼네팅 결과 
  - 192.168.0.0/22  ← 하나의 큰 네트워크로 결합됨

### 전제 조건

- 연속된 네트워크여야 함 (정렬된 블록 단위)
- CIDR(Classless Inter-Domain Routing)을 지원하는 라우팅 프로토콜 사용 
- 예: RIPv2, OSPF, EIGRP 등 
- 비지원: RIPv1 (클래스풀 라우팅)

### 주의 사항

- 중간에 빠진 네트워크가 있으면 슈퍼네팅 불가
- 전체 네트워크 블록의 2의 제곱 크기 단위로만 결합 가능
