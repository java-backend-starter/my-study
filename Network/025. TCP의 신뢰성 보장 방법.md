# TCP의 신뢰성 보장 방법

---

## 1. Window Size (윈도우 크기)

### 윈도우 크기란?

- 윈도우 크기(Window Size)는 송신자가 수신자로부터 매번 확인 응답(ACK)을 받지 않고도 연속해서 전송할 수 있는 최대 데이터 양을 의미한다.
- 수신 측의 버퍼 용량에 따라 결정되며, 3-way Handshake 과정에서 서로의 버퍼 크기를 교환해 초기 윈도우 크기를 설정한다


### 윈도우 크기의 필요성

- 세그먼트를 하나씩 보내고 매번 응답을 기다리는 방식은 비효율적입니다.
- 연속해서 세그먼트를 보내고 나중에 응답을 받는 방식이 효율적이지만, 수신 측의 버퍼를 초과하면 데이터 유실 가능성이 있다.
- 윈도우 크기는 송신자가 수신자의 버퍼 용량을 고려해 보낼 수 있는 데이터 양을 조절하는 흐름 제어(Flow Control) 수단이다.

---

## 2. ACK (Acknowledgment)

- 수신 측이 데이터를 정상적으로 받았음을 송신 측에 알리는 신호다.
- ACK 번호는 수신 측이 받은 마지막 바이트의 순서 번호에 1을 더한 값이다.
- 송신 측은 해당 ACK 번호 이전의 데이터가 모두 수신되었다고 간주하고, 그 다음 데이터를 전송한다.

---

## 3. SEQ (Sequence Number)

- TCP 세그먼트에 포함된 데이터의 순서 번호이다.
- 데이터는 여러 세그먼트로 나뉘어 전송되며, 수신 측은 이 순서 번호를 기반으로 데이터의 순서를 맞춘다.
- SEQ 번호는 각 세그먼트의 시작 바이트 위치를 식별하기 위해 사용된다.
