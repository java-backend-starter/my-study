# L4(전송 계층) vs. L7(애플리케이션 계층) 로드 밸런싱

---

## 1. L4 로드밸런서 정의

- 작동 계층: 전송 계층 (Layer 4)
- 주요 프로토콜: TCP, UDP 
- 로드밸런싱 기준: IP 주소, 포트 
- 장점: 빠른 처리 속도 (패킷 헤더만 분석)
- 단점: 애플리케이션 계층 정보 활용 불가 
- 사용 사례: 실시간 트래픽 처리 (온라인 게임, 스트리밍 서비스 등)

---

## 2. L7 로드밸런서 정의

- 작동 계층: 애플리케이션 계층 (Layer 7)
- 주요 프로토콜: HTTP, HTTPS 
- 로드밸런싱 기준: 요청 내용 (URL, 헤더, 쿠키 등)
- 장점: 다양한 기능 및 유연성 제공 (요청 분석, 캐싱, 압축 등)
- 단점: 상대적으로 느린 처리 속도 (애플리케이션 계층 정보 분석 필요)
- 사용 사례: 애플리케이션 레벨 로드밸런싱 (웹 서비스, API 게이트웨이, CDN 등)

---

## 3.  L4와 L7 로드밸런서 비교

| 항목               | L4 로드밸런서            | L7 로드밸런서              |
|--------------------|-------------------------|---------------------------|
| 작동 계층          | 전송 계층 (Layer 4)      | 애플리케이션 계층 (Layer 7) |
| 주요 프로토콜      | TCP, UDP                | HTTP, HTTPS               |
| 로드밸런싱 기준    | IP 주소, 포트           | 요청 내용 (URL, 헤더, 쿠키 등) |
| 처리 속도          | 빠름                    | 상대적으로 느림           |
| 기능 및 유연성     | 제한적                  | 다양하고 유연함           |

---

## 4. 장단점 및 사용 사례

### L4 로드밸런서

- 장점: 빠른 응답 시간 
- 단점: 기능 및 유연성이 제한적 
- 사용 사례: 실시간 트래픽이 중요한 서비스 (온라인 게임, 스트리밍 서비스 등)

### L7 로드밸런서

- 장점: 다양한 기능과 유연성 제공 
- 단점: 느린 처리 속도 
- 사용 사례: 애플리케이션 레벨 로드밸런싱이 필요한 서비스 (웹 서비스, API 게이트웨이, CDN 등)

---

## 5. 적절한 활용

- L4 로드밸런서: 실시간 트래픽 처리에 적합, 빠른 응답이 필요한 서비스에 사용 
- L7 로드밸런서: 애플리케이션 레벨에서 세밀한 트래픽 분산 및 다양한 기능이 필요한 서비스에 사용 
- 이 두 기술을 적절히 활용하는 것이 중요하며, 웹 개발자는 상황에 맞게 선택해야 합니다.
