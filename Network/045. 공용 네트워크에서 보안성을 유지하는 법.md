# 공용 네트워크에서 보안성을 유지하는 법

---

## 0. 개요

- 공용 Wi-Fi는 접근성과 편리함을 제공하지만, 사용자 정보를 노리는 다양한 사이버 공격의 주요 표적이 되기 쉽다. 
- 본 문서는 공용 네트워크에서 발생할 수 있는 보안 위협을 이해하고, 실질적인 대응 전략과 보안 수칙을 정리하여 사용자 및 개발자의 보안 인식을 높이는 데 목적이 있다.

## 1. 공용 네트워크의 주요 보안 위협

- 중간자 공격 (MITM, Man-In-The-Middle)
  - 공격자가 사용자와 서버 사이의 통신을 가로채어 데이터를 탈취하거나 조작.
- 패킷 스니핑 
  - 암호화되지 않은 네트워크 패킷을 분석하여 비밀번호, 쿠키 등 민감 정보를 도청.
- 세션 하이재킹 
  - 사용자 인증 이후 발급된 세션 ID를 탈취하여 사용자로 가장.
- 악성 액세스 포인트 (Evil Twin)
  - 공격자가 만든 가짜 Wi-Fi에 사용자가 연결하게 하여 정보를 탈취.

## 2. 공용 네트워크에서의 보안 유지 방법

### VPN 사용

- SSL 기반 터널링을 통해 네트워크 상의 데이터 전송을 암호화.
- IP 주소 은폐 및 DNS 누수 방지** 기능으로 위치 정보 및 사용 기록 보호.
- 무료 VPN보다는 신뢰할 수 있는 유료 VPN 사용을 권장.

### HTTPS 접속 확인

- 웹사이트 주소가 `https://`로 시작하는지 확인.
- TLS(Transport Layer Security) 핸드셰이크 과정을 통해 공개키 기반 암호화로 데이터 보호.
- SSL 인증서는 공인 인증기관(CA)에서 발급되며, 인증서 위조 방지를 위한 신뢰 체계가 존재.

### 자동 연결 비활성화

- 기기의 Wi-Fi 설정에서 자동 연결 기능을 비활성화하여 가짜 AP 연결 방지.

### 2단계 인증 활성화

- 비밀번호 외에 인증 앱, 문자, 생체 인식 등의 보조 인증 방식 추가로 보안 강화.

### 민감한 작업은 피하기

- 공용 네트워크에서는 인터넷 뱅킹, 쇼핑, 로그인 등 민감한 정보 입력을 피함.

### 방화벽 및 보안 소프트웨어 사용

- 실시간 감시 및 악성코드 차단 기능으로 사전 위협 차단.

### 운영체제 및 앱 최신 상태 유지

- 보안 취약점을 통한 공격을 막기 위해 정기적인 업데이트 필수.

## 3. 위협별 대응 매칭표

| 위협 유형    | 설명                                   | 대응 방안                               |
|--------------|----------------------------------------|-----------------------------------------|
| 중간자 공격   | 사용자와 서버 사이에서 데이터를 탈취          | VPN 사용, HTTPS 접속 확인                 |
| 세션 하이재킹  | 세션 쿠키를 탈취하여 사용자로 위장           | 2단계 인증, 세션 타임아웃 설정               |
| 악성 AP      | 가짜 와이파이를 통한 정보 탈취               | 자동 연결 비활성화, 네트워크 신뢰 설정       |
| 패킷 스니핑   | 암호화되지 않은 패킷을 가로채기              | HTTPS, VPN, SSL Pinning 적용           |
