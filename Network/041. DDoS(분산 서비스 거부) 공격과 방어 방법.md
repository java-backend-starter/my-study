# DDoS 공격과 방어 방법

---

## 1. DDoS 공격이란?

- DDoS 공격은 수백에서 수천 대의 컴퓨터나 IoT 기기 등 다수의 장치를 이용하여 특정 서버나 네트워크에 대량의 트래픽을 유발함으로써 서비스의 정상적인 운영을 방해하는 공격이다. 
- 공격자는 타겟 시스템의 리소스를 소모시켜 서비스 중단을 초래하며, 이로 인해 금전적 손실, 브랜드 가치 하락, 사용자 이탈 등의 피해가 발생할 수 있다.

---

## 2. 주요 DDoS 공격 유형

- DDoS 공격은 다양한 방식으로 이루어지며, 주요 유형은 다음과 같다.

### 볼륨 기반 공격 (Volume-based)

- 대량의 트래픽을 생성하여 네트워크 대역폭을 소진시키는 공격이다. 
- UDP Flood: 대량의 UDP 패킷을 전송하여 시스템의 패킷 처리 리소스를 소모시킨다. 
- ICMP Flood: 수많은 ICMP 패킷을 전송하여 네트워크 리소스를 소모하게 한다. 
- Spoofed-packet Flood: 정상적인 IP 주소를 위조하여 대량의 패킷을 전송, 필터링을 어렵게 만든다.

### 프로토콜 공격 (Protocol-based)
   
- 네트워크 프로토콜의 취약점을 악용하여 시스템 리소스를 소모시키는 공격이다. 
- SYN Flood: 수많은 SYN 요청을 보내 세션 연결을 시도, 리소스를 소모시킨다. 
- Ping of Death: 비정상적이거나 큰 ICMP 패킷을 전송하여 시스템을 충돌시킨다. 
- Smurf 공격: 브로드캐스트 주소로 ICMP Echo를 보내 네트워크에 과부하를 유발한다.

### 애플리케이션 계층 공격 (Application-layer)
   
- 웹 서버나 DNS 서버 등 애플리케이션의 취약점을 악용하여 리소스를 고갈시키는 공격이다.
- HTTP Flood: 대량의 HTTP 요청을 보내 웹 서버의 리소스를 소모시킨다. 
- Slowloris: 부분적인 HTTP 요청을 보내 세션을 오래 유지시켜 리소스를 소모시킨다. 
- DNS Query Flood: 대량의 DNS 쿼리를 보내 DNS 서버에 과부하를 유발한다.
- 

### 인프라 계층 공격 (Infrastructure-layer)
   
- 라우터, 스위치 등 네트워크 인프라의 취약점을 악용하여 전체 네트워크를 마비시키는 공격이다.
- UDP, ICMP, SYN Flood DDoS 공격도 포함되며, 네트워크의 기본 구성 요소들을 표적으로 삼아 공격하기에 타겟 시스템의 피해 규모가 거대할 수 있다.

## 3. ? DDoS 방어 전략

- 효과적인 DDoS 방어를 위해서는 다음과 같은 전략이 필요하다

### 트래픽 모니터링 및 분석
   
- 네트워크 트래픽을 지속적으로 모니터링하여 비정상적인 패턴을 탐지하고, Flow 분석, 이상 탐지(Anomaly detection), 기준선 설정(Baselining) 등을 통해 잠재적인 공격을 조기에 식별한다.

### 네트워크 필터링 및 속도 제한

- 악성 IP 주소를 차단하거나, 요청 비율을 제한하는 Rate limiting, 특정 지역의 트래픽을 제한하는 Geo-blocking 등을 통해 공격 트래픽을 효과적으로 차단한다.

### 애플리케이션 계층 보안
   
- 웹 애플리케이션 방화벽(WAF) 등을 활용하여 애플리케이션 계층에서의 공격을 방어하고, 리버스 프록시를 통해 DDoS 트래픽을 필터링한다.

### 하이브리드 클라우드 기반 방어

- 온프레미스 DDoS 방어 하드웨어(A10, Radware 등)와 클라우드 기반 DDoS 방어 서비스(Cloudflare 등)를 결합하여 대규모의 정교한 DDoS 공격에도 효과적으로 대응한다.

### 대응 프로세스 수립
   
- 중요 인프라와 시스템을 파악하고 우선순위를 부여하여 DDoS 공격으로부터 선제적으로 보호하며, 책임과 역할, 커뮤니케이션 채널, 에스컬레이션 절차를 포함한 전체적인 대응 프로세스를 수립하고 유지한다.

---

## 4. DDoS 방어의 4단계

- 탐지: 트래픽 모니터링을 통해 비정상적인 패턴을 조기에 탐지한다. 
- 대응: 사전에 정의된 방어 정책을 활성화하고, 네트워크 설정을 조정하여 공격에 대응한다. 
- 필터링: 공격 트래픽과 정상 트래픽을 구분하여 정상적인 트래픽만을 허용한다. 
- 복구: 공격 종료 후 시스템을 정상 상태로 복구하고, 향후 공격에 대비한 보안 강화 조치를 시행한다.
