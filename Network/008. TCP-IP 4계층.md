# TCP/IP 4계층

---

## 0. 개요

- TCP/IP 4계층 모델은 네트워크 통신을 4개의 계층으로 나누어 각 계층이 특정한 역할을 수행하도록 설계된 구조입
- 이러한 계층화는 각 기능이 독립적으로 동작하여 유지보수에 용이한 장점을 제공

---

## 1. 네트워크 연결 계층 (Network Access Layer)

- 역할
  - 물리적인 데이터 전송을 담당
  - 같은 네트워크 내에서 장치 간 신뢰성 있는 데이터 전송을 보장
  - 물리적 주소인 MAC 주소를 사용하여 장비 간 통신을 수행
  - 본적인 오류 검출과 프레임화를 담당
- 데이터 단위: 프레임 (Frame)
- 전송 주소: MAC 주소
- 예시 프로토콜 및 장비: 이더넷(Ethernet), 브리지(Bridge), 스위치(Switch)

---

## 2. 인터넷 계층 (Internet Layer)

- 역할
  - 데이터의 논리적 주소 지정과 경로 설정을 통해 서로 다른 네트워크 간의 데이터 전송을 가능하게 함
  - IP 주소를 활용하여 장치를 식별하고
  - 라우팅을 통해 최적의 경로로 데이터를 전달
- 데이터 단위: 패킷 (Packet)
- 전송 주소: IP 주소
- 예시 프로토콜 및 장비: IP, ARP, ICMP, 라우터(Router)

---

## 3. 전송 계층 (Transport Layer)

- 역할
  - 통신 노드 간의 데이터 전송의 신뢰성을 보장
  - 데이터가 적절한 애플리케이션에 정확히 전달되도록 해서 종단 간(end-to-end) 신뢰성을 확
- 데이터 단위: 세그먼트 (Segment)
- 전송 주소: 포트 번호
- 예시 프로토콜: TCP(연결 지향형, 신뢰성 보장), UDP(비연결형, 신뢰성 미보장)

---

## 4. 응용 계층 (Application Layer)

- 역할
  - 사용자와 가장 가까운 계층으로, 응용 프로그램 간의 데이터 송수신을 담당
  - 웹 브라우저, 이메일 클라이언트 등 다양한 응용 프로그램이 이 계층에서 동작
- 데이터 단위: 데이터(Data) 또는 메시지(Message)
- 예시 프로토콜: HTTP, FTP, DNS, SMTP

---

## 5. 데이터 전송 과정

- 송신 측: 데이터는 각 계층을 거치며 캡슐화(encapsulation) 과정을 통해 하위 계층으로 전달됨
- 수신 측: 캡슐화된 데이터는 역순으로 각 계층에서 비캡슐화(decapsulation) 과정을 거쳐 원래의 데이터를 복원함 
- 이러한 구조는 네트워크 통신의 효율성과 신뢰성을 높이는 데 기여함.

---

## 6. 비교

- TCP/IP 모델과 OSI 7계층 모델은 네트워크 통신을 계층별로 구조화하여 이해를 돕는 표준 모델 
- 두 모델은 계층 수, 명칭, 기능에서 차이가 있지만, 네트워크 통신 과정을 체계적으로 설명하는 공통점이 존재

### TCP/IP vs OSI 7계층 비교표

| TCP/IP 계층                         | OSI 7계층                         |
|------------------------------------|----------------------------------|
| 응용 계층 (Application Layer)       | 응용 계층 (Application Layer)     |
|                                    | 표현 계층 (Presentation Layer)    |
|                                    | 세션 계층 (Session Layer)         |
| 전송 계층 (Transport Layer)         | 전송 계층 (Transport Layer)       |
| 인터넷 계층 (Internet Layer)        | 네트워크 계층 (Network Layer)     |
| 네트워크 인터페이스 계층            | 데이터 링크 계층 (Data Link Layer) |
| (Network Interface Layer)          | 물리 계층 (Physical Layer)        |

### 주요 차이점

1. 계층 구조
    - TCP/IP는 4계층 구조이며, OSI는 7계층으로 더 세분화되어 있음
    - TCP/IP의 응용 계층은 OSI의 응용, 표현, 세션 계층을 포함
    - TCP/IP의 네트워크 인터페이스 계층은 OSI의 데이터 링크와 물리 계층을 포함
2. 개발 시기 및 목적
    - TCP/IP는 실제 인터넷 프로토콜로, 실용적 목적
    - OSI 7계층은 이론적인 표준 모델로 이해와 설계를 위한 목적
3. 프로토콜 구성
    - TCP/IP는 실제 운영되는 프로토콜 중심 (HTTP, TCP, IP 등)
    - OSI는 기능 중심의 추상적 모델로 각 계층의 역할을 설명

### 활용 방안

- TCP/IP는 실제 네트워크 통신에서 사용되며, 실무 중심의 모델
- OSI 7계층은 각 계층의 역할과 기능을 이론적으로 이해하는 데 유용
- 두 모델은 상호 보완적으로 활용 가능하며, 네트워크 문제 해결 및 설계 시 함께 고려됨
