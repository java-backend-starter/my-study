# 로드 밸런서(Load Balancer)

---

## 1. 로드 밸런서란?

- 로드 밸런서는 사용자와 서버 사이에 위치하여 여러 서버에 트래픽을 분산시켜 성능을 최적화하고 안정적인 서비스를 제공한다. 
- 이는 단일 서버의 한계를 극복하고 시스템의 확장성과 가용성을 높이기 위한 핵심 구성 요소이다.

---

## 2. 로드 밸런싱의 주요 이점

- 가용성 향상: 서버 장애나 과부하 시에도 서비스 중단 없이 운영 가능하며, 상태 확인을 통해 문제를 사전에 방지한다. 
- 확장성 확보: 트래픽 증가에 따라 서버를 유연하게 추가하거나 제거할 수 있어 시스템 확장이 용이히다. 
- 보안성 강화: DDoS와 같은 악성 트래픽을 분산하거나 차단하여 보안을 향상시킨다. 
- 성능 개선: 지능적인 부하 분산으로 응답 속도를 높이고 지연 시간을 줄인다.

---

## 3.  로드 밸런서의 종류

### L4 로드 밸런서 (전송 계층)
   
- 기능: IP 주소와 포트 정보를 기반으로 트래픽을 분산한다. 
- 장점: 단순하고 빠르며, 리소스 소모가 적다. 
- 특징: 클라이언트와 서버 간의 초기 연결만 로드 밸런서를 거치며, 이후 통신은 직접 이루어진다.

### L7 로드 밸런서 (애플리케이션 계층)
   
- 기능: HTTP 헤더, 쿠키 등 애플리케이션 레벨의 데이터를 분석하여 트래픽을 분산한다. 
- 장점: 세분화된 트래픽 제어가 가능하며, 보안 기능이 강화되어 있다. 
- 단점: 복잡한 처리로 인해 리소스 소모가 크다. 
- 특징: 모든 트래픽이 로드 밸런서를 통해 전달된다.

---

## 4. 로드 밸런싱 알고리즘

- 라운드 로빈 (Round Robin): 요청을 순차적으로 각 서버에 분배한다. 서버 성능이 유사할 때 효과적이다. 
- 가중 라운드 로빈 (Weighted Round Robin): 서버의 성능에 따라 가중치를 부여하여 요청을 분배한다. 
- 최소 연결 방식 (Least Connection): 현재 연결 수가 가장 적은 서버에 요청을 할당한다. 세션 시간이 불규칙할 때 유리하다. 
- DNS 라운드 로빈: DNS를 통해 여러 서버의 IP를 순환하여 응답한다. 구현이 간단하지만 캐싱 문제로 트래픽이 특정 서버에 집중될 수 있다.
