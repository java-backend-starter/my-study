# 물리 계층

---

## 1. 개요

- OSI 7계층 중 최하위 계층
- 컴퓨터 간 물리적 연결 담당
- 데이터를 전기 신호(디지털/아날로그)로 변환해 전송

---

## 2. 데이터 전송 과정

1. 문장을 0과 1의 비트로 변환
2. 비트를 전기 신호(디지털)로 변환 → 랜 카드가 수행
3. 케이블을 통해 전기 신호 전송
4. 수신 측에서 전기 신호를 다시 비트로 복원

---

## 3. 랜 카드 (LAN Card)

- 비트를 전기 신호로 변환하는 장치
- 데이터가 컴퓨터 밖으로 나가는 출발점 역할

---

## 4. 케이블

- 역할: 전기 신호를 다른 장치로 전달
- 종류:
    - 동축 케이블: 중앙 구리선을 통한 신호 전송
    - 꼬임선 케이블 (UTP, FTP, STP)
    - 광섬유 케이블 (설명 생략)

---

## 5. 장비: 리피터 vs 허브

### 리피터 (Repeater)

- 전기 신호를 증폭하는 장치
- 신호 감쇠 방지
- 1:1 통신만 가능
- 현재는 잘 사용되지 않음

### 허브 (Hub)

- 리피터 기능 포함한 상위 호환 장비
- 데이터를 여러 포트로 복제 및 분배
- 여러 컴퓨터와 연결 가능
- 특정 대상 지정 없이 모든 장치에 데이터 전송
    - → 더미 허브(Dummy Hub) 라고도 부름

---

## 6. 이더넷

### 이더넷(Ethernet)이란?

- 이더넷은 로컬 영역 네트워크(LAN)에서 가장 널리 사용되는 통신 규격.
- TCP/IP 모델의 네트워크 인터페이스 계층에 해당하며, 호스트 간 프레임(Frame) 단위로 데이터를 주고받음.
- 주로 L2 스위치를 통해 연결된 동일 네트워크 내 장치 간 통신을 담당.

### 통신 구조 및 방식

- 각 호스트는 이더넷 인터페이스를 갖고 있으며, 이를 통해 데이터 송수신이 이루어짐.
- L2 스위치는 MAC 주소 기반으로 목적지 포트를 식별하여 프레임을 전달.
- 스위치는 데이터를 변형하지 않고 전달만 하는 역할을 하며, 프레임 내부 내용은 건드리지 않음.

### 이더넷 규격

| 규격        | 속도     | 케이블 유형     | 설명                       |
|-------------|----------|------------------|----------------------------|
| 10BASE-T    | 10Mbps   | UTP              | 기본 이더넷                |
| 100BASE-TX  | 100Mbps  | UTP              | 패스트 이더넷(Fast Ethernet) |
| 1000BASE-T  | 1Gbps    | Cat 5e 이상      | 기가비트 이더넷            |
| 10GBASE-T   | 10Gbps   | Cat 6a 이상      | 10기가비트 이더넷          |

- BASE는 Baseband 방식 사용을 의미
- T는 Twisted-pair 케이블을 의미

### MAC 주소 (Media Access Control)

- MAC 주소는 네트워크 카드(NIC)마다 부여되는 고유한 48비트 식별자입니다.
- 16진수로 표현되며, 예: 00:1A:2B:3C:4D:5E
- 프레임 전송 시 다음 두 주소가 필수로 포함:
    - 목적지 MAC 주소
    - 출발지 MAC 주소

### 전송 매체 (Transmission Medium)

- 가장 일반적인 전송 매체는 UTP(Unshielded Twisted Pair) 케이블
- 8개의 선이 4쌍으로 꼬여 있으며, 노이즈 감소와 간섭 방지를 위한 구조
- 커넥터는 RJ-45가 표준으로 사용

### 이더넷 프레임 구조

| 필드        | 설명                                                   |
|-------------|--------------------------------------------------------|
| Destination | 목적지 MAC 주소                                        |
| Source      | 출발지 MAC 주소                                        |
| Type        | 상위 프로토콜 식별 (예: IPv4는 0x0800)                |
| Payload     | 실제 데이터 (예: IP 패킷)                              |
| FCS         | 오류 검출용 Frame Check Sequence (CRC 기반)           |

### 네트워크 토폴로지

- 현재 이더넷 네트워크는 스타형(Star Topology) 구조를 사용합니다.
- 중앙에 L2 스위치를 두고 각 장비가 별도로 연결됨
- 과거에는 버스형(Bus Topology)이 사용되었으며, 하나의 케이블에 여러 호스트가 연결되어 있었습니다.

### 전송 제어 방식: CSMA/CD

- Carrier Sense Multiple Access with Collision Detection
- 주로 버스형 이더넷 환경에서 사용되던 충돌 제어 방식
- 주요 동작 절차:
    - Carrier Sense: 채널이 비었는지 확인
    - Multiple Access: 여러 호스트가 동일 채널 공유
    - Collision Detection: 전송 중 충돌 발생 시 감지
    - Backoff Algorithm: 랜덤 시간 후 재전송 시도
- 현재의 스위치 기반 이더넷에서는 충돌이 없기 때문에 CSMA/CD는 사실상 사용되지 않음

## 7. 광섬유(Optical Fiber)

- 빛을 이용해 정보를 전송하는 유리 또는 플라스틱 섬유
- 높은 대역폭과 낮은 신호 손실로 유선 통신의 핵심 매체로 사용됨
- 레이저 기술의 발전과 함께 보급됨

### 원리

- 광섬유는 코어(Core)와 클래딩(Cladding)으로 구성
- 코어는 높은 굴절률, 클래딩은 낮은 굴절률을 가짐
- 빛은 두 매질 사이에서 전반사되어 전송
- 전송 방식에 따라 다음과 같이 분류됨:
    - 단일 모드(Single-mode): 한 경로로만 전송
    - 다중 모드(Multi-mode): 여러 경로로 전송

### 장점

- 높은 대역폭으로 많은 데이터 전송 가능
- 낮은 감쇠율로 장거리 전송에 유리
- 전자기 간섭(EMI)에 강함
- 전기를 사용하지 않아 감전 위험 없음
- 정보 누출 위험이 낮아 보안에 유리
- 가볍고 유연하여 설치가 용이
- 에너지 효율이 높고 수명이 김

### 단점

- 초기 설치 및 유지보수 비용이 높음
- 물리적 충격에 약하여 취급 주의 필요
- 연결 및 수리에 고도의 기술 요구
- 전력 공급이 중단되면 통신 불가
- 장거리 전송 시 중계기 필요

### 활용 분야

- 통신 및 네트워크: 초고속 인터넷, 전화, IPTV 등
- 의료: 내시경, 레이저 수술 장비 등
- 센서 기술: 온도, 압력 등 다양한 센서 응용
- 국방: 군용 통신망, 정밀 유도 무기 시스템 등
- 산업 자동화: 공장 제어 및 모니터링 시스템
- 조명 및 디스플레이: 인테리어 조명, 예술 설치 등

### 전망

- 5G, IoT, 스마트 시티 등 신기술과의 결합을 통해 더욱 중요해질 전망
- 지속적인 기술 발전으로 비용 절감 및 성능 향상 기대
