# 전송 계층

---

## 1. 정의

- 전송 계층(Transport Layer)은 OSI 7계층 중 4계층에 해당하며, 송신자와 수신자 간의 논리적인 통신을 제공하는 계층
- 이 계층은 서로 다른 호스트에 있는 애플리케이션 프로세스들 사이의 통신을 다룸
- 논리적 통신: 실제 네트워크의 복잡함을 숨기고, 프로세스 간에 직접 연결된 것처럼 보이게 함

---

## 2. 구조

1. 애플리케이션 메시지를 세그먼트(segment)로 분할
2. 각 세그먼트에 트랜스포트 계층 헤더 추가
3. 이 세그먼트를 네트워크 계층에 전달
4. 네트워크 계층에서는 데이터그램으로 캡슐화하여 전송
   - 전송 계층은 종단 시스템(End System)에서만 구현됨
   - 주고받는 데이터 단위는 세그먼트
   - 네트워크 계층에서 사용하는 단위는 데이터그램

---

## 3. 역할

### 프로세스 간 통신

- 전송 계층은 호스트가 아닌 프로세스 간의 통신을 지원
- 포트 번호를 통해 각 프로세스를 식별

### 세분화 및 재조립

- 메시지를 작은 세그먼트로 분리하여 전송
- 수신 측에서는 이를 재조립함

###신뢰성 있는 전송 (TCP)

- 순서 보장
- 오류 검출 및 복구
- 중복 제거

### 연결 관리

- TCP는 3-way 핸드셰이크를 통해 연결 설정
- 4-way 핸드셰이크로 연결 종료

### 비신뢰성 전송 (UDP)

- 오류 복구 및 순서 보장을 하지 않음
- 대신 속도가 빠름

---

## 4. 전송 계층과 네트워크 계층의 차이

| 전송 계층 | 네트워크 계층 |
|-----------|----------------|
| 프로세스 간 통신 제공 | 호스트 간 통신 제공 |
| 종단 시스템에서만 구현 | 라우터 포함 네트워크 전역에 구현 |
| 포트 번호 사용 | IP 주소 사용 |

비유:
- 네트워크 계층: 집에서 집까지 편지를 보내는 일 (주소 기반)
- 전송 계층: 집 안에서 누가 편지를 받는지 지정하는 일 (포트 기반)

---

## 5. 전송 계층의 주요 프로토콜

### TCP (Transmission Control Protocol)

- 연결지향형
- 신뢰성 보장
- 흐름 제어, 혼잡 제어, 오류 복구 포함

### UDP (User Datagram Protocol)

- 비연결형
- 빠르지만 신뢰성 없음
- 실시간 서비스에 적합 (예: VoIP, 스트리밍)
