# 메모리 보호 기술

---

## 1. 메모리와 CPU의 협업

- 모든 프로그램은 메모리에 올라와야 실행된다. 
- 메모리에는 실행에 필요한 프로그램과 데이터가 존재하며, CPU와 협업하여 작업이 이루어진다. 
- 메모리의 기본 소자 크기는 1bit로, 0 또는 1이 저장된다. 
- 그러나 소자 하나로는 데이터 표현이 불가능하여, 메모리는 바이트 단위로 지정되며 데이터 읽기나 쓰기 시 워드 단위로 이동한다.
- 예를 들어, 32bit 대역폭을 가진 버스에서는 1bit 단위로 데이터를 이동할 필요는 없다.

### 메모리의 종류

#### 램(RAM)

- 휘발성 메모리: 전력 공급이 끊기면 데이터가 사라진다. 
- DRAM (Dynamic RAM): 일정 시간마다 재생이 필요하며, 속도는 느리지만 비용이 저렴하다. 
- SRAM (Static RAM): 전력이 공급되는 동안 데이터가 사라지지 않아 빠르지만 가격이 비싸다.
- SDRAM (Synchronous DRAM): 클록틱에 맞춰 데이터를 저장하는 동기 DRAM. 
- 비휘발성 메모리: 전력 공급이 끊겨도 데이터를 보관할 수 있다. 
- 플래시 메모리: USB 드라이버, 디지털카메라 등에서 사용되며, 저장 가능한 횟수가 제한적이다. 
- SSD: 하드디스크 대신 사용되는 저장장치로, 빠른 속도와 내구성, 저전력을 제공한다.

#### 롬(ROM)

- 롬은 데이터를 한 번만 저장할 수 있고 전력이 끊겨도 데이터를 보관할 수 있다. 
- 롬의 종류는 다음과 같다:
  - 마스크 롬: 한 번만 데이터를 저장할 수 있는 ROM. 
  - PROM (Programmable ROM): 한 번은 데이터를 저장할 수 있으며, 특수 기기를 사용해 프로그램할 수 있다. 
  - EPROM (Erasable Programmable ROM): 데이터를 여러 번 지우고 쓸 수 있다. 
  - DDR SDRAM
    - DDR (Double Data Rate SDRAM)
      - 기존 SDRAM의 대역폭을 두 배로 늘린 메모리로, 데이터를 클록틱 한 번에 2개의 워드로 저장할 수 있다. 
      - DDR2, DDR3, DDR4: DDR SDRAM의 발전된 형태로, 대역폭이 계속해서 증가하고 있다.

## 2. 메모리 보호

- 메모리 보호는 여러 작업이 동시에 실행되는 환경에서 중요하다. 
- 운영체제와 사용자 프로그램 간의 메모리 보호를 위해 CPU는 두 가지 레지스터를 사용한다. 
  - 경계 레지스터: 현재 작업의 메모리 시작 주소를 저장. 
  - 한계 레지스터: 작업이 차지하는 메모리 크기(마지막 주소)를 저장.
- 이 두 레지스터는 하드웨어적으로 메모리 범위를 체크하여, 작업이 자신의 메모리 영역을 벗어나지 않도록 보호한다. 
- 만약 범위를 벗어나면 메모리 오류와 관련된 인터럽트가 발생하고, 운영체제가 해당 오류를 처리한다.
