# 프로세스와 스레드

---

## 1. 프로세스(Process)

- 실행 중인 프로그램의 인스턴스
- 운영체제가 자원(CPU, 메모리 등)을 할당
- 독립적인 메모리 공간 사용
- 다른 프로세스와 메모리 공유 안 됨
- IPC (파이프, 소켓, 공유 메모리 등)을 통해 통신 (복잡함)

---

## 2. 스레드(Thread)

- 프로세스 내의 작업 흐름 단위
- 여러 스레드가 하나의 프로세스 자원을 공유
- 메모리 공유를 통해 통신이 간단하고 전환이 빠름
- 한 스레드 오류 시 전체 프로세스에 영향 줄 수 있음

---

## 3. 프로세스 vs 스레드

| 항목         | 프로세스(Process)                                   | 스레드(Thread)                                   |
|--------------|-----------------------------------------------------|--------------------------------------------------|
| 메모리       | 독립적 메모리 공간                                   | 메모리 공유 (스택 제외)                          |
| 자원 할당    | 운영체제가 새 자원 할당                              | 프로세스 자원 공유                               |
| 실행 속도    | 전환 느림                                            | 전환 빠름                                        |
| 통신 방법    | IPC 사용 (복잡)                                      | 메모리 공유 (간단)                               |
| 오버헤드     | 큼                                                   | 적음                                             |
| 안정성       | 독립적 (다른 프로세스에 영향 없음)                  | 오류가 같은 프로세스 내 다른 스레드에 영향 가능 |

---

## 4. 예시

- 프로세스: 웹 브라우저, 텍스트 편집기, 파일 탐색기 등 독립 실행 프로그램
- 스레드: 웹 브라우저 내에서 웹 페이지 로딩, 사용자 입력 처리, 음악 재생 등 동시 실행 작업
