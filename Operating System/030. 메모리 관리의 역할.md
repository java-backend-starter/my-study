# 메모리 관리의 역할

---

## 1. 메모리란?

- 메모리는 프로그램 실행 시 필요한 주소와 정보를 저장하고 가져다 사용할 수 있게 만드는 공간으로, 주로 RAM을 의미합니다.

--- 

## 2.  메모리 관리의 필요성

- 프로세스 간 메모리 격리: 각각의 프로세스는 독립된 메모리 공간을 갖고, 운영체제나 다른 프로세스의 메모리 공간에 접근할 수 없습니다.
- 운영체제의 역할: 운영체제는 메모리 영역에 대한 접근 제약을 받지 않으며, 메모리 관리의 책임을 집니다.
- 멀티프로그래밍 환경: 한정된 메모리를 효율적으로 사용하기 위해 운영체제의 메모리 관리 방법이 중요해졌습니다.

---

## 3. 메모리 관리 방법

### 연속 메모리 할당

- 기법: 프로세스를 메모리에 연속적으로 할당하는 방식입니다.
- 단점: 할당과 제거를 반복하다 보면 Scattered Holes(작은 빈 공간)가 생겨나고, 이로 인한 외부 단편화가 발생합니다.

### 페이징(Paging)

- 기법: 프로세스의 논리적 메모리를 고정 크기의 페이지로 나누고, 물리적 메모리의 프레임에 비연속적으로 할당하는 방식입니다.
- 장점: 외부 단편화를 해결할 수 있습니다.
- 단점: 내부 단편화가 발생할 수 있습니다.

### 세그먼테이션(Segmentation)

- 기법: 프로세스의 논리적 메모리를 서로 다른 크기의 세그먼트로 나누고, 이를 물리적 메모리에 할당하는 방식입니다.
- 장점: 내부 단편화를 해결할 수 있습니다.
- 단점: 외부 단편화가 발생할 수 있습니다.

### 메모리 풀(Memory Pool)

- 기법: 필요한 크기와 개수만큼 미리 메모리를 할당받아 놓고, 필요할 때마다 사용하고 반납하는 방식입니다.
- 장점: 메모리 할당과 해제가 잦은 경우에 효과적이며, 단편화를 줄일 수 있습니다.
- 단점: 미리 할당해놓은 메모리를 사용하지 않으면 낭비될 수 있습니다.
