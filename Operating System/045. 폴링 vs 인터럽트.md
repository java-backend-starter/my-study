# 폴링 vs 인터럽트

---

## 1. 인터럽트(Interrupt) 방식

- '새치기', '중단'이라는 의미를 가진다.
- CPU가 프로그램을 실행하는 중에 주변 장치의 입출력 요청이나 하드웨어 문제, 프로그램 예외 등이 발생했을 때 CPU에게 알리는 방식이다.
- 인터럽트 핀에 신호가 들어오면,
    1. 현재 실행 중인 주소를 스택에 저장하고
    2. 인터럽트 서비스 루틴(ISR: Interrupt Service Routine)을 실행한다.
    3. 루틴이 끝나면 스택에 저장해둔 주소로 복귀하여 원래 작업을 이어간다.

---

## 2. 폴링(Polling) 방식

- CPU가 주변 장치의 상태를 주기적으로 감시하고 체크하는 방식이다.
- 장치의 레지스터를 주기적으로 읽어 상태를 파악해야 한다.
- 주기적인 감시를 위해 `while`문 등 반복문 형태로 구현된다.
- 이 작업 동안은 다른 장치의 체크가 어렵고, CPU 리소스를 많이 소모한다.

---

## 3. 정리
- 둘 다 외부 이벤트에 따라 특정 작업을 수행하는 방식이다.
- 둘 다 CPU와 입출력 장치 간 통신을 위한 방법이다.

---

## 4. 비교 표

| 항목            | 인터럽트 방식                           | 폴링 방식                             |
|-----------------|----------------------------------------|--------------------------------------|
| 동작 방식       | 외부 장치가 CPU에 작업 요청             | CPU가 주기적으로 상태 확인             |
| CPU 자원 사용   | 필요할 때만 처리 → 효율적                | 계속 상태 체크 → 비효율적              |
| 응답 속도       | 이벤트 발생 시 즉시 대응 가능            | 체크 주기에 따라 늦어질 수 있음          |
| 코드 구조       | 이벤트 발생 시 ISR 호출                  | 반복문으로 상태 주기적 체크              |
| 사용 예시       | 키보드 입력 처리, 하드웨어 오류 감지 등   | 간단한 타이머 체크, 장치 상태 주기적 확인 |
