# 멀티 프로세스와 멀티 스테드

---

## 1. 멀티 프로세스 (Multi Process)

### 정의 및 개념

- 하나의 프로그램이 여러 개의 독립된 프로세스로 실행됨 
- 각 프로세스는 자신만의 메모리 공간을 가지고 독립적으로 동작 
- 일반적으로 하나의 부모 프로세스가 여러 자식 프로세스를 생성함 
- 예시: 크롬 브라우저의 각 탭

### 멀티 프로세스의 장점

- 프로그램 안전성 
  - 각 프로세스는 격리되어 동작 
  - 한 프로세스의 오류가 전체에 영향을 주지 않음 
  - 예: 크롬 탭 중 하나가 오류나도 다른 탭은 정상 작동
- 프로그램 병렬성
  - 여러 CPU 코어를 이용해 병렬 작업 가능 
  - 이미지 처리, 비디오 인코딩 등에서 활용 가능 
  - 다만, 멀티 스레드보다 느리고 비효율적일 수 있음
- 시스템 확장성
  - 새로운 기능을 별도 프로세스로 추가하기 쉬움 
  - 유지보수 및 테스트가 용이

### 멀티 프로세스의 단점

- 컨텍스트 스위칭 오버헤드 
  - 프로세스 간 전환 시, 메모리 상태 전체를 교체해야 하므로 무겁고 느림

- 자원 공유 비효율성 
  - 메모리 공간이 분리되어 있음 
  - IPC(파이프, 소켓 등)를 사용한 복잡한 통신 필요

---

## 2. 멀티 스레드 (Multi Thread)

### 정의 및 개념

- 하나의 프로세스 안에서 여러 스레드가 동작 
- 메모리 공간과 자원을 공유함 
- 프로세스 내에서 병렬 작업을 수행할 수 있게 해 줌 
- 예시: 게임에서 UI, 사운드, 입력 처리를 동시에 수행

### 멀티 스레드의 장점

- 자원 공유 효율성 
  - 모든 스레드가 동일한 메모리 공간 공유
  - 데이터 공유와 통신이 빠름
- 응답성 향상 
  - 사용자 요청, 백그라운드 작업 등을 분리하여 UI가 버벅이지 않도록 함
- 멀티 코어 프로세서 활용 
  - CPU 코어가 많을수록 더 많은 스레드를 병렬 처리 가능 
  - 성능 향상 가능

### 멀티 스레드의 단점

- 안정성 문제 
  - 하나의 스레드 오류로 전체 프로세스가 종료될 수 있음
- 동기화로 인한 성능 저하 
  - 공유 자원 접근 시 락 등의 동기화 필요
  - 병목 현상 발생 가능
- 데드락 (교착 상태)
  - 두 스레드가 서로 자원을 기다리며 무한 대기할 수 있음
- 컨텍스트 스위칭 오버헤드 
  - 스레드 간 전환에도 오버헤드가 존재 
  - 프로세스보다는 덜하지만 무시할 수 없음
- 디버깅의 어려움 
  - 실행 순서가 일정하지 않아 디버깅이 어렵고 복잡함
- 운영체제의 지원 필요 
  - 운영체제에 따라 스레드의 효율성과 기능 지원에 차이가 있음
