# 라운드 로빈

---

## 1. 개요 및 동작 방식

- 라운드 로빈(Round Robin, RR): '순환 순서 방식'
- 선점형 스케줄링 알고리즘 중 가장 단순
- 타임 슬라이스(Time Slice) 동안 작업 수행
- 작업 미완료 시 준비 큐의 맨 뒤로 이동
- FCFS와 유사하나, 타임 슬라이스의 존재가 차이

---

## 2. 성능 및 평균 대기 시간 예시

- 타임 슬라이스: 10ms
- 프로세스 도착 시점 및 실행 시간:
    - P1: 0ms
    - P2: 3ms
    - P3: 6ms
- 평균 대기 시간 계산:
    - P1: 0 + 19 + 8 = 27ms
    - P2: 7 + 19 = 26ms
    - P3: 14ms
      → 총 대기 시간 = 67ms
      → 평균 = 22.33ms

※ 콘보이 효과 완화에 유리함

---

## 3. 타임 슬라이스와 문맥 교환

- 문맥 교환(Context Switching): 프로세스 전환 시 발생하는 오버헤드
- 동일 평균 대기 시간이라도 RR은 문맥 교환 비용 때문에 FCFS보다 비효율적일 수 있음

---

## 4. 타임 슬라이스 크기의 영향

- [타임 슬라이스가 큰 경우]
  - 장점: 문맥 교환 적음
  - 단점: FCFS와 유사, 반응성 떨어짐
- [타임 슬라이스가 작은 경우]
  - 장점: 사용자 입장에서 프로그램이 동시에 실행되는 듯한 느낌
  - 단점: 문맥 교환이 자주 발생 → 실제 작업 시간 줄어듦 → 시스템 성능 저하
- 타임 슬라이스는 작게 설정하되, 문맥 교환 오버헤드를 고려해 적정 크기로 조절 필요

---

## 5. 실제 운영체제 적용 예

- UNIX 시스템:
    - 타임 슬라이스 약 100ms
    - 조절 범위: 10~200ms
