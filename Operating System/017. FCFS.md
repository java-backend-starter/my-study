# FCFS

---

## 1. 개념

- 준비 큐에 도착한 순서대로 CPU 할당
- 비선점형 방식
- FIFO(First In First Out) 큐 방식
- 예: 손님을 한 팀씩 받는 레스토랑

---

## 4. 동작 방식

- 프로세스가 도착한 순서대로 실행
- 실행 중인 프로세스가 종료되어야 다음 프로세스 실행 가능
- 모든 프로세스는 동일한 우선순위

---

## 3. 성능 평가 (평균 대기 시간 예시)

- P1 (도착: 0ms, 실행: 30ms) → 대기 시간: 0ms
- P2 (도착: 3ms, 실행: 18ms) → 대기 시간: 30 - 3 = 27ms
- P3 (도착: 6ms, 실행: 9ms)  → 대기 시간: 48 - 6 = 42ms
- 평균 대기 시간: (0 + 27 + 42) / 3 = 23ms

---

## 4. 단점 및 평가

- 콘보이 효과 (Convoy Effect): 긴 작업이 CPU 점유 시 다른 프로세스들이 장시간 대기
- 입출력 요청 시 CPU가 유휴 상태가 되는 시간 많음 
  - 시분할 시스템에서는 해결 가능, 일괄 처리 시스템에서는 비효율
- 구조는 단순하고 공정하지만 시스템 효율성은 낮음
