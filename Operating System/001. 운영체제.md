# 운영체제

---

## 1. 개요

- 운영체제(Operating System, OS)는 컴퓨터 시스템의 하드웨어 및 소프트웨어 자원을 효율적으로 관리하여 사용자가 컴퓨터를 편리하고 효과적으로 사용할 수 있도록 돕는 시스템 소프트웨어
- 하드웨어와 사용자 또는 다른 소프트웨어 간의 중개자 역할을 수행

---

## 2. 운영체제의 주요 역할 및 목적

### 역할

1. 자원 관리: CPU, 메모리, 입출력 장치 등의 하드웨어 자원과 프로세스, 파일 등의 소프트웨어 자원을 효율적으로 관리한다.
2. 자원 보호: 프로그램이나 사용자가 다른 프로그램의 데이터나 중요한 파일에 무단으로 접근하거나 삭제하지 못하도록 보호한다.
3. 인터페이스 제공: 하드웨어와 사용자 간의 인터페이스를 제공하여 컴퓨터 시스템을 편리하게 사용할 수 있도록 지원한다.

### 목적

- 응답 시간(Turnaround Time): 작업을 의뢰한 후 결과를 얻기까지의 시간
- 처리 능력(Throughput): 일정 시간 동안 시스템이 처리하는 작업의 양
- 사용 가능도(Availability): 시스템을 얼마나 빠르고 지속적으로 사용할 수 있는지의 정도
- 신뢰도(Reliability): 주어진 문제를 얼마나 정확하게 처리하는지의 정도

---

## 3. 운영체제의 종류

1. Windows: 마이크로소프트에서 개발한 데스크톱 운영체제로, 사용자 친화적인 인터페이스를 제공한다.
2. Linux: 리누스 토르발스가 개발한 오픈 소스 운영체제로, 다중 사용자와 다중 작업을 지원하며 서버 및 개발 환경에서 널리 사용된다.
3. Unix: 벨 연구소에서 개발된 멀티태스킹과 다중 사용자 방식을 지원하는 운영체제로, 현대 운영체제의 기반이 되었다.

---

## 4. 운영체제의 유형

- 일괄 처리 시스템(Batch Processing System): 유사한 작업들을 일정량 또는 일정 시간에 묶어서 처리하는 방식
- 다중 프로그래밍 시스템(Multiprogramming System): 하나의 CPU와 주기억 장치를 이용하여 여러 개의 프로그램을 동시에 처리하는 방식
- 시분할 시스템(Time Sharing System): 시간을 분할하여 여러 사용자에게 컴퓨터 자원을 번갈아가며 할당하는 방식
- 실시간 시스템(Real-Time System): 단말기의 요청을 즉시 처리하여 결과를 반환하는 시스템
- 다중 처리 시스템(Multiprocessing System): 여러 대의 CPU와 하나의 주기억 장치를 이용하여 여러 개의 프로그램을 동시에 처리하는 방식
- 분산 컴퓨팅(Distributed Computing): 네트워크로 연결된 여러 대의 컴퓨터의 처리 능력을 이용하여 복잡한 계산 문제를 해결하는 방식

---

## 5. 운영체제의 주요 기능

- 프로세스 관리: 동시에 실행되는 여러 프로세스를 효율적으로 관리한다.
- 파일 관리: 파일의 생성, 수정, 삭제, 백업 등을 수행한다.
- 네트워크 관리: 네트워크 서비스의 성능과 오류를 분석하고 품질을 관리한다.
- 메인 메모리 관리: 메모리 사용을 추적하고 프로세스에 따라 메모리를 할당한다.
- 보조 스토리지 관리: 스토리지 할당, 여유 공간 관리, 디스크 스케줄링 등을 수행한다.
- 입출력 장치 관리: 버퍼 캐싱 시스템과 장치 드라이버를 제공하여 입출력 장치를 효율적으로 관리한다.
- 보안 관리: 시스템을 보호하고 오류로 인한 손상을 방지한다.
- 명령어 해석 시스템: 사용자와 시스템 간의 인터페이스를 제공하여 명령어를 해석하고 실행한다.
