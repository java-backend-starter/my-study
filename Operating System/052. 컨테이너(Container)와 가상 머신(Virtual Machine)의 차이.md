# 컨테이너(Container)와 가상 머신(Virtual Machine)의 차이

---

## 1. 컨테이너와 가상 머신(VM) 개요

- 컨테이너와 가상 머신(VM)은 시스템을 격리하여 패키징하는 방법이다. 
- 둘의 가장 큰 차이점은 격리되는 구성 요소에 있으며, 이는 규모와 이식성에 영향을 준다.

---

## 2. 컨테이너란?

- 정의: 애플리케이션 실행에 필요한 모든 요소(코드, 라이브러리 등)를 하나로 묶은 소프트웨어 단위.
- 특징 
  - 메가바이트(MB) 단위로 가벼움 
  - 하이퍼바이저 없이 프로세스 단위로 분리 
  - 빠르고 민첩함 
  - 뛰어난 이식성: 개발자 노트북, 데이터 센터, 클라우드, 엣지 어디서나 동일하게 작동
- 주요 기술: Docker 
- 오케스트레이션: 수백 개의 컨테이너를 관리하기 위해 Kubernetes, OpenShift 등을 사용

---

## 3. 가상 머신(VM)이란?

- 정의: 독립된 인스턴스에서 운영 체제를 실행하며, 물리적 컴퓨터를 에뮬레이션하는 기술.
- 특징 
  - 하이퍼바이저를 통해 물리 리소스를 가상화 
  - 하나의 서버에서 여러 VM 운영 가능 
  - 자체 운영체제(OS)를 가짐 
  - 리소스 집약적이고 유연성, 멀티테넌시 제공
- 활용 
  - 2000년대부터 통합 및 비용 절감 목적으로 확산 
  - 클라우드 컴퓨팅 초기 기반 
  - GPU 등 고비용 리소스 최적화

---

## 4. 작동 방식

### 가상화

- 하이퍼바이저가 물리적 리소스를 분할하여 VM에 할당 
- 대규모 OS 풋프린트로 인해 단일 애플리케이션 실행에는 과함

### 컨테이너

- 애플리케이션, 라이브러리, 종속성을 패키징한 컨테이너 이미지 사용 
- 커널 공유 
- 작은 크기 덕분에 유연하고 탄력적

---

## 5. 어느 기술을 사용해야 하나?

### 컨테이너가 적합한 경우

- 소형 인스턴스 필요 
- 클라우드 네이티브 애플리케이션 빌드 
- 마이크로서비스 패키징 
- DevOps, CI/CD 통합 
- 다양한 IT 환경에 확장 필요

### VM이 적합한 경우

- 기존, 레거시, 모놀리식 워크로드 운영 
- 위험한 개발 사이클 격리 
- 네트워크, 서버, 데이터 인프라 프로비저닝 
- 다른 종류의 OS 실행(Linux 위에 Unix 등)
