# 보안과 관련된 OS의 접근 제어 방식

---

## 1. 접근제어(Access Control) 정의
   
- 정보 시스템에서 적절한 권한을 가진 사용자만 특정 자원이나 정보에 접근할 수 있도록 통제하는 기술 또는 절차 
- 접근통제(Access Control)라고도 하며, 정보보호의 3요소(기밀성, 무결성, 가용성) 중 기밀성(confidentiality)을 중심으로 구성된다.

---

## 2. 접근의 종류 (CRUD 기반)
   
- Read: 정보 조회 
- Create: 정보 생성 
- Update: 정보 수정 
- Delete: 정보 삭제 
- 각 접근 유형에 대해 권한이 부여되어야 접근 가능

---

## 3. 접근제어의 일반적 과정
   
- 접근 요청(Request): 사용자가 시스템이나 정보에 접근을 시도함 
- 접근 허용 또는 거부(Grant/Deny): 시스템이 권한을 검토한 후 허용 또는 차단 결정

---
## 4. 접근제어의 3단계 절차

### 식별 (Identification)
   
- 사용자가 자신이 누구인지 밝히는 과정 
- 시스템은 해당 ID를 인식하고 기록함 
- 예시: 사용자명(ID), 계정 번호, 주민등록번호, 학번 등

### 인증 (Authentication)
   
- 사용자가 제시한 식별 정보가 실제 본인인지 확인하는 과정 
- 예시: 비밀번호, OTP, 스마트카드, 생체 인증(지문, 홍채 등)

### 인가 (Authorization)
   
- 인증된 사용자가 어떤 자원에 어느 수준으로 접근 가능한지 권한을 부여하는 과정 
- 예시: ACL(Access Control List), RBAC(Role-Based Access Control)

### 정리

```
접근제어 = 인증(Authentication) + 인가(Authorization)
```

---

## 5. 사용자 인증의 유형

| 구분                 | 설명                   | 예시                              | 단점                                |
|----------------------|------------------------|------------------------------------|-------------------------------------|
| Something You Know   | 사용자가 알고 있는 것 | 패스워드, PIN                      | 분실, 도용 가능                     |
| Something You Have   | 사용자가 소유한 것    | 스마트카드, OTP 토큰, 신분증       | 분실 가능                           |
| Something You Are    | 사용자의 생체적 특성  | 지문, 홍채, 음성, 얼굴             | 생체정보 유출 시 복구 어려움       |
| Something You Do     | 행동 기반 특성        | 서명 습관, 타이핑 속도, 걸음걸이   | 구현 복잡                           |

## 6. 접근통제 유형

### 임의적 접근 통제 (DAC: Discretionary Access Control)
   
- 자원의 소유자가 접근 권한을 직접 제어 
- 사용자가 누구인지에 따라 허용 여부 결정 
- 유연하지만 보안 취약 
- 예시: UNIX/Linux의 파일 권한(chmod), Windows NTFS 권한

### 강제적 접근 통제 (MAC: Mandatory Access Control)
   
- 시스템이 미리 정의된 보안 정책에 따라 접근을 강제 제어 
- 사용자나 관리자도 권한을 변경할 수 없음 
- 기밀성 중심, 군사 시스템 등에서 사용 
- 보안 레벨이 등급별로 지정됨 (예: Secret, Top Secret)

### 비임의적 접근 통제 (NDAC: Non-Discretionary Access Control)
   
- 접근 권한이 직무나 역할 등 사용자 특성에 따라 자동 부여 
- DAC보다 통제가 엄격하며, MAC보다 유연함 
- 주로 조직 기반 시스템에 사용 
- 세부 유형
  - 역할 기반 접근 제어(RBAC): 사용자의 역할(Role)에 따라 권한 부여 
  - 직무 기반 접근 제어(ABAC): 직무(Job function)에 따라 권한 부여 
  - 래티스 모델(Lattice-based Access Control): 보안 레벨과 권한이 격자구조로 구성됨

---

## 7. 접근 통제 해석

| 유형                         | 설명                                               |
|------------------------------|----------------------------------------------------|
| MAC                          | 규칙 기반, 보안 등급 비교, 기밀성 중시            |
| DAC                          | 사용자가 소유 권한 설정, 유연하지만 보안 약함     |
| RBAC                         | 역할 기반, 업무에 따라 권한 부여                  |
| Rule-Based Access Control    | 사전에 설정된 규칙에 따라 통제                    |

## 8. 접근통제 보안모델

### 상태머신 모델 (State Machine Model)
   
- 가장 기초적이고 이론적인 모델 
- 시스템이 항상 허용된 상태만 유지해야 한다는 관점

### 벨-라파둘라 모델 (Bell-LaPadula Model)
   
- 최초의 수학적 기밀성 모델 
- 기밀성 중심, 군사/정부 시스템에 적합 
- 보안 정책
  - No Read Up (NRU): 자신보다 높은 등급 정보 읽기 금지 
  - No Write Down (NWD): 자신보다 낮은 등급에 쓰기 금지
- 단점: 무결성 보장 안됨 (Blind Write 문제)

### 비바 모델 (Biba Model)
   
- 무결성 중심 모델 
- 벨-라파둘라 모델을 반대로 적용 
- 보안 정책 
  - No Read Down (NRD): 자신보다 낮은 무결성 등급 정보 읽기 금지 
  - No Write Up (NWU): 자신보다 높은 무결성 등급에 쓰기 금지
- 사용 사례: 회계, 은행 등에서 데이터 신뢰성 확보

### 클락-윌슨 모델 (Clark and Wilson Model)
   
- 상업 환경에 적합, 무결성 보장 중시 
- 보안 원칙 
  - Well-Formed Transactions: 정해진 방식으로만 데이터 변경 
  - Separation of Duties: 업무 분리로 불법 행위 방지 
  - Double-entry Bookkeeping: 이중 기록 원칙(회계)

### 래티스 모델 (Lattice-Based Model)
   
- 주체/객체에 보안 등급 부여 
- 정보 흐름이 위계 구조에 따라 제한됨 
- 모든 정보는 상위/하위 계층에만 흐를 수 있음 
- MAC 구현에 적합
