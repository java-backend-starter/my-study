# 커널

---

## 1. 커널의 정의

- 커널은 운영체제의 핵심 부분
- 하드웨어와 응용 프로그램 사이에서 인터페이스를 제공하며 컴퓨터 자원을 관리
- 항상 메모리에 상주하며, 사용자와의 직접적인 상호작용은 지원하지 않음

---

## 2. 자원 관리 역할

- 커널은 물리적 하드웨어 자원을 추상화하여 관리
  - CPU → 프로세스(Task)
  - 메모리 → 페이지(Page) 또는 세그먼트(Segment)
  - 디스크 → 파일(File)
  - 네트워크 장치 → 소켓(Socket)
- 이를 통해 각 사용자가 하드웨어를 독점적으로 사용하는 것처럼 느끼게 함

---

## 3. 주요 기능

- 프로세스 관리: CPU를 프로세스로 추상화하여 제공
- 메모리 관리: 메모리를 페이지 또는 세그먼트로 추상화하여 제공
- 파일 시스템 관리: 디스크를 파일로 추상화하여 제공
- 네트워크 관리: 네트워크 장치를 소켓으로 추상화하여 제공
- 디바이스 드라이버 관리: 각종 외부 장치에 대한 접근 관리
- 인터럽트 처리: 인터럽트 핸들링
- 입출력 통신 관리

---

## 4. 인터페이스로서의 역할

- 커널은 사용자 공간(User Space)과 커널 공간(Kernel Space) 사이에서 시스템 콜 인터페이스(System Call Interface)를 통해 사용자 요청을 하드웨어에 전달하는 역할

---

## 5. 커널과 운영체제의 관계

- 커널은 운영체제의 핵심 구성 요소이며, 여기에 사용자 애플리케이션과 유틸리티가 추가되면 전체 운영체제가 완성

## 6. 커널의 유형

### 모놀리식 커널(Monolithic Kernel)

- 애플리케이션을 제외한 모든 시스템 서비스를 커널이 직접 처리하는 방식
- 처리 속도가 빠르지만 커널 크기가 크고 오류 발생 시 전체 시스템에 영향을 줄 수 있음

### 마이크로 커널(MicroKernel)

- 핵심 서비스만을 커널에 포함하고 나머지 서비스는 개별 서버 형태로 존재
- 가볍고 기능 추가 및 수정이 용이하지만, 시스템 복잡도가 증가할 수 있음
