# 프로세스 우선순위 스케줄링

---

## 1. 개요 및 동작 방식

- 각 프로세스는 중요도에 따라 우선순위(priority)를 가짐
- 우선순위가 높은 프로세스가 먼저 CPU 할당받음
- 다양한 기준으로 우선순위 설정 가능 (예: 작업 시간 짧을수록 높음)
- FCFS + 우선순위 개념 적용 = SJF와 유사한 결과

---

##2. 우선순위 스케줄링 방식

- [비선점형 방식]
  - SJF (Shortest Job First): 작업 시간이 짧을수록 우선순위 ↑
  - HRN (Highest Response-ratio Next):
    - 우선순위 = (대기 시간 + 실행 시간) / 실행 시간
    - 대기 시간이 길거나 실행 시간이 짧을수록 우선순위 ↑
- [선점형 방식]
  - SRT (Shortest Remaining Time):
    - 남은 작업 시간이 짧을수록 우선순위 ↑
- 비선점형/선점형 모두 우선순위 스케줄링에 포함될 수 있음

---

## 3. 우선순위 정책 유형

- 고정 우선순위 알고리즘
  - 프로세스 생성 시 우선순위 고정
  - 단순하지만 시스템 변화 반영 어려움
- 변동 우선순위 알고리즘
  - 일정 주기마다 우선순위 재계산
  - 구현 복잡하지만 시스템 상황 반영 가능

---

## 4. 우선순위 스케줄링의 단점 및 평가

- 공정성 문제
  - 준비 큐 순서를 무시하고 높은 우선순위만 우선 실행
  - 낮은 우선순위 프로세스가 영원히 실행되지 못하는 '아사(starvation)' 발생 가능
- 오버헤드 증가
  - 우선순위를 주기적으로 계산, 갱신해야 하므로 처리 비용 발생
- 그러나 우선순위는 "효율성"보다 "중요도"에 기반
  - 예: 커널 프로세스가 일반 프로세스보다 항상 우선해야 함 
  - 시스템 효율성과 사용자 만족도 간의 균형 필요
