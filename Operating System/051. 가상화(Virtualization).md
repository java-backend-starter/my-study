# 가상화(Virtualization)

---

## 1. 가상화(Virtualization)

- 하나의 물리적 서버에서 여러 운영체제(OS)와 애플리케이션을 동시에 실행할 수 있도록 하는 소프트웨어 기술. 
- 물리적 컴포넌트(하드웨어 장치)를 논리적 객체로 추상화하여, 하나의 장치를 여러 개처럼, 여러 장치를 하나처럼 사용자에게 제공하는 방식.
- 주요 가상화 대상: CPU, 메모리, 스토리지, 네트워크 등.

---

## 2. 가상화의 기대 효과
   
- 자원 사용률 극대화. 
- 서버 통합 및 관리 비용 절감. 
- 분산 처리 능력 향상. 
- 시스템 확장성과 유연성 강화.

---

## 3. 가상화의 역사
   
### 1960년대 
- IBM 메인프레임에서 최초로 가상화 시도. 
- 서버 성능은 무어의 법칙에 따라 지속적으로 향상되었음.

### 문제 인식 
- 서버 1대당 1개 애플리케이션을 구동 → 서버 자원을 100% 활용하지 못함. 
- 서버 비용은 높은데 활용률은 낮아 비효율 발생.

### 가상화 등장 

- 물리 서버 하나를 여러 개의 논리적 서버처럼 사용하여 효율 극대화. 
- 1970년대 다양한 벤더(Vendor) 솔루션 등장.

### 상용화 시대 (2000년대)

- 2001년: x86 서버용 최초 상용 솔루션인 VMware ESX 발표. 
- 2003년: Citrix에서 오픈소스 기반 Xen 솔루션 발표.

---

## 4. 가상화 종류
   
- 가상화는 대상에 따라 다음과 같이 나뉜다. 

### 서버 가상화 (Server Virtualization)

- 가상화 개념의 시초. 
- 서버의 하드웨어 자원을 논리적으로 나누어 다수의 가상머신(VM)에서 독립적으로 운영체제를 실행. 
- 하이퍼바이저(Hypervisor)를 이용하여 제어. 
  - 하드웨어의 물리적 리소스를 가상머신(VM)들에게 제공. 
  - VM과 하드웨어 간의 입출력(I/O) 명령을 처리.

#### 대표 하이퍼바이저 

- VMware vSphere 
- Microsoft Hyper-V 
- KVM, Xen 등

### 데스크톱 가상화 (Virtual Desktop Infrastructure, VDI)

- 데이터 센터의 서버에서 가상 데스크톱(PC 환경)을 중앙 관리하는 기술. 
- 사용자 PC는 실제 컴퓨팅 작업을 수행하지 않고, 중앙 서버의 가상 환경에 접속해 화면만 전달받아 사용.
#### 특징 
- 동일 데이터와 애플리케이션을 사용하기 위해 동일 PC가 필요 없음. 
- 네트워크를 통해 어디서든 접속하여 작업 가능. 
- 사용자 데이터는 서버에 저장되므로 데이터 유출 위험 감소.

#### 성능 좌우 요소 

- 네트워크 속도 (인터넷 연결 품질)
- 스토리지 성능 (서버 저장장치 속도)

#### VDI 도입 이유 

- 보안 강화 
  - 업무망과 인터넷망을 물리적으로 분리하여 망분리 보안 구현 가능. 
- 관리 효율성 
  - 동일 OS 및 애플리케이션을 일괄 설치/복제/회수 가능. 
- 스마트 오피스 구현 
  - 사내/외부 어디서나 자신의 데스크톱 환경에 접속 가능.
  - 유연한 근무 환경 제공.

### 애플리케이션 가상화 (Application Virtualization)

- 애플리케이션을 중앙 서버에 설치하고, 클라이언트에는 실행 화면과 인터페이스만 전송하여 사용하게 하는 기술. 
- 애플리케이션은 클라이언트에 직접 설치되지 않음. 
- 그러나 마치 설치된 것처럼 동작.

#### 장점 

- 관리의 효율성 
  - 수백~수천대 PC에 대해 일괄 배포/업데이트 가능. 
  - 다양한 운영체제(OS)와 시스템 환경에서도 설치 없이 사용 가능. 
- 응용프로그램 간 충돌 방지 
  - 하나의 애플리케이션 업데이트가 다른 애플리케이션에 영향을 주지 않음. 
- 보안 강화 
  - 클라이언트 디바이스에 데이터가 저장되지 않아 유출 위험 감소. 
  - 액세스 과정에 대한 통제가 가능.

#### 주요 도입 사례 

- 기업 내부 시스템 보호 
- 대규모 기업 내 애플리케이션 통합 관리
