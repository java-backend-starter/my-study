# SJF

---


## 1. 개념

- 실행 시간이 가장 짧은 작업부터 CPU 할당
- 비선점형 스케줄링
- 최단 작업 우선 스케줄링 (SPF라고도 함)
- 예: 정수기 줄에서 큰 물통 대신 양해를 구하고 작은 컵 먼저 받기

---

## 2. 동작 방식

- 준비 큐의 프로세스 중 작업 시간이 짧은 순서로 실행
- FCFS의 콘보이 효과 완화 → 시스템 효율성 향상

---

## 3. 성능 평가 (예시)

- P1 (도착: 0ms, 실행: 30ms) → 대기: 0ms
- P3 (도착: 6ms, 실행: 9ms)  → 대기: 30 - 6 = 24ms
- P2 (도착: 3ms, 실행: 18ms) → 대기: 39 - 3 = 36ms
- 평균 대기 시간: (0 + 24 + 36) / 3 = 20ms

---

## 4. 단점 및 문제점

- 종료 시간 예측 어려움 
  - 사용자 입력/상호작용 등으로 정확한 실행 시간 예측 불가
- 공정성 문제 (아사 현상 발생 가능)
  - 작업 시간이 짧은 프로세스가 계속 들어오면, 긴 작업은 계속 연기됨 (무한 봉쇄)

---

## 5. 해결 방안 (제한적)
- 프로세스가 예상 작업 시간을 직접 제출 
  - 정확도 부족 및 악의적인 속임 가능성 있음
- 에이징(Aging)
  - 순서를 양보할수록 '나이'가 올라가 일정 기준이 되면 강제 실행 
  - 하지만 나이 기준 정하기가 까다로움

6. ? 결론
- 효율성은 높지만, 예측 어려움과 아사 현상 때문에 실제 운영체제에서 사용 빈도 낮음
