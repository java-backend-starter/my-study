# 멀티 레벨 큐

---

## 1. 다단계 큐 스케줄링 (Multilevel Queue Scheduling)

### 개요

- 다단계 큐 스케줄링은 우선순위에 따라 여러 개의 큐를 사용하는 방식이다. 
- 각 프로세스는 우선순위에 따라 특정 큐에 배치되며, 큐 내에서 라운드 로빈 방식으로 스케줄링된다. 
- 각 큐의 우선순위가 고정되어 있으며, 상위 큐가 끝날 때까지 하위 큐의 작업은 실행되지 않는다.

### 특징

- 우선순위에 따라 프로세스가 큐에 배치된다.
- 각 큐는 라운드 로빈 방식으로 스케줄링된다.
- 우선순위가 높은 큐의 프로세스가 먼저 실행된다.
- 높은 우선순위의 프로세스가 실행될 때, 낮은 우선순위의 프로세스는 대기 상태로 남는다.

## 2. 다단계 피드백 큐 스케줄링 (Multilevel Feedback Queue Scheduling)

### 개요

- 다단계 피드백 큐 스케줄링은 다단계 큐 스케줄링의 단점을 보완한 방식이다.
- 프로세스가 CPU를 할당받으면 우선순위가 낮아져서 다른 큐로 이동한다. 이를 통해 낮은 우선순위 프로세스의 실행을 연기하는 문제를 해결한다.

### 특징

- 각 프로세스는 실행 후 우선순위가 낮아져서 하위 큐로 이동한다.
- 타임 슬라이스의 크기는 우선순위에 따라 달라지며, 우선순위가 낮은 프로세스일수록 타임 슬라이스가 커진다.
- 마지막 큐에 있는 프로세스는 FCFS(선입선출) 방식으로 처리된다.
- 우선순위가 낮은 프로세스는 무한대의 타임 슬라이스를 받아 CPU를 빼앗기지 않고 끝까지 작업을 마친다.

### 요약

- 다단계 큐 스케줄링은 고정된 우선순위에 따라 큐를 나누고, 상위 큐가 먼저 실행되는 방식이다.
- 다단계 피드백 큐 스케줄링은 프로세스의 우선순위가 CPU 사용 후 낮아져 하위 큐로 이동하며, 우선순위에 따라 타임 슬라이스 크기가 달라진다.
  - 이를 통해 낮은 우선순위 프로세스의 실행 기회를 확대하고, 마지막 큐에서 FCFS 방식으로 처리된다.
