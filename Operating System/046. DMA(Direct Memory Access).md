# DMA(Direct Memory Access)

---

## 1. DMA(Direct Memory Access)란?

- CPU의 개입 없이 메모리와 주변 장치 간 데이터를 직접 전송하는 기술이다. 
- 데이터 전송 효율을 높이고, CPU의 작업 부하를 줄이는 데 중요한 역할을 한다.

### 구성 요소

- DMA Controller : 데이터 전송을 제어하는 전용 하드웨어이다. 
- Bus : 메모리와 주변 장치 간 데이터를 전달하는 통로이다. 
- Interrupt : 데이터 전송이 완료되었음을 CPU에 알리는 메커니즘이다.

### 작동 과정

- 주변 장치가 DMA 컨트롤러에 데이터 전송을 요청한다. 
- DMA 컨트롤러가 CPU로부터 버스 제어권(Bus Master)을 얻는다. 
- DMA 컨트롤러가 메모리와 주변 장치 간에 직접 데이터를 전송한다. 
- 전송이 완료되면 DMA 컨트롤러가 인터럽트를 통해 CPU에 작업 완료를 알린다.

### 장점

- CPU가 데이터 전송 작업에서 벗어나 계산, 프로세스 제어 등에 집중할 수 있다. 
- 대량 데이터 처리 시 전송 속도가 빨라지고, 병렬 처리가 가능해진다.

### 응용 분야

- 디스크 I/O (HDD, SSD 등 데이터 읽기/쓰기)
- 네트워크 데이터 처리 (고속 네트워크 패킷 처리)
- 멀티미디어 데이터 처리 (오디오, 비디오 장치 데이터 전송)
- 임베디드 시스템 (센서 데이터 수집 및 처리)

### 한계점

- DMA 설정이 복잡하다. 
- CPU와 DMA가 동시에 같은 메모리 영역에 접근할 경우 충돌이 발생할 수 있다. 
- DMA 컨트롤러 추가로 인해 하드웨어 비용이 발생한다.

## 2. DMA 간접 실습 아이디어

### I/O 효율성 고려 시스템 설계 실습

- 파일 업로드/다운로드 최적화 (Java NIO vs IO 비교 실험)
- 대규모 로그 처리 시스템을 구현하고 비동기 처리 구조를 적용한다.

### 비동기 프로세스와 멀티스레드 실습

- 멀티스레드 기반 데이터 처리 프로그램을 작성한다. 
- Spring WebFlux를 사용해 비동기 데이터 처리 API를 구현한다.

### 메모리 효율성 고려 캐싱 실습

- Ehcache나 Redis를 활용해 API 결과를 캐싱한다. 
- 캐싱 전후 성능을 비교하고 데이터 동기화 문제를 해결한다.

### 대규모 데이터 전송 시나리오 실습

- REST API를 설계해 JSON/CSV 형식의 대량 데이터를 전송한다. 
- 데이터 압축(Gzip)과 스트리밍을 적용해 성능을 높인다. 
- Spring Batch를 이용해 대규모 데이터 이동 및 변환 작업을 처리한다.

### 성능 분석 및 튜닝 실습

- Java Mission Control, VisualVM 등을 사용해 메모리, 스레드 상태를 분석한다. 
- Spring Boot Actuator와 Prometheus를 활용해 시스템 성능을 모니터링한다.

### 하드웨어 연관 간접 실습

- JNI(Java Native Interface)를 학습해 Java에서 C 라이브러리를 호출하는 방법을 익힌다. 
- NIO와 BIO 파일 입출력 방식을 비교해 성능 차이를 분석한다.
