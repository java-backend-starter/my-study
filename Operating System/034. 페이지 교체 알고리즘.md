# 페이지 교체 알고리즘

---

## 1. 페이지 교체 알고리즘의 개요

- 페이지 부재(Page Fault): 프로세스가 요구한 페이지가 메모리에 없을 때 발생.
- 메모리가 가득 찼을 때: 새로운 페이지를 메모리에 올리기 위해 기존 페이지를 스왑 영역으로 내보내야 함.
- 목적: 페이지 교체 알고리즘은 어떤 페이지를 스왑 영역으로 내보낼지 결정하여, 시스템 성능을 최적화하고 페이지 부재를 줄이는 역할.

---

## 2. 페이지 교체 알고리즘의 종류

| 알고리즘               | 특징                                                                 |
|------------------------|----------------------------------------------------------------------|
| 무작위(Random)          | 무작위로 페이지를 선택하여 교체. 구현이 가장 간단하지만 성능은 낮음.  |
| FIFO(First In, First Out) | 가장 먼저 메모리에 들어온 페이지를 교체. 구현 간단하지만, 최적의 성능은 아님. |
| 최적(Optimal)           | 미래의 페이지 접근을 예측하여 가장 덜 사용될 페이지를 교체. 성능은 가장 우수하나, 실제 구현은 불가능. |
| LRU(Least Recently Used) | 가장 오래 전에 사용된 페이지를 교체. 최적 알고리즘에 근접하지만, 구현이 복잡. |
| LFU(Least Frequently Used) | 가장 적게 사용된 페이지를 교체.                                   |
| NUR(Not Used Recently)  | 페이지가 최근에 사용되지 않은 경우 교체.                            |
| 2차 기회 페이지 교체    | FIFO의 변형. 페이지가 참조되지 않으면 교체 대상으로, 참조된 페이지는 재처리하여 교체를 지연시킴. |
| 시계 알고리즘(Clock)    | FIFO와 유사하지만, 각 페이지에 접근 비트가 있어 참조되지 않은 페이지를 효율적으로 교체. |

---

## 3. 페이지 교체 알고리즘의 성능 평가 기준

- 페이지 부재 횟수: 페이지 교체 알고리즘 성능 비교의 기본 기준. 페이지 부재가 적을수록 성능이 우수.
- 평균 대기 시간: 페이지를 요청한 후, 실제 작업이 시작될 때까지의 시간.
- 전체 작업 시간: 전체 작업을 처리하는데 소요된 시간.
- 유지 비용: 알고리즘이 사용하는 계산 자원(메모리, CPU 시간 등). 성능이 우수하더라도, 계산 비용이 많이 들거나 메모리 요구가 크면 비효율적일 수 있음.

---

## 4. 페이지 교체 알고리즘 적용 예시

- 메모리 접근 순서: 특정 메모리 접근 순서를 기반으로 알고리즘을 테스트하고 비교.
- 프레임 수: 예시에서는 물리 메모리가 3개의 프레임을 가지고 있는 상황을 가정.

---

## 5. 알고리즘 간 비교

- 가장 간단한 알고리즘: 무작위, FIFO.
- 성능은 좋으나 구현이 어려운 알고리즘: 최적.
- 최적에 근접하는 알고리즘: LRU, LFU, NUR.
- FIFO 변형 알고리즘: 2차 기회, 시계 알고리즘.
