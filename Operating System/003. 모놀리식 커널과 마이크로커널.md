# 모놀리식 커널과 마이크로커널

---

## 1. 모놀리식 커널 (Monolithic Kernel)

- 구조: 프로세스 관리, 메모리 관리, 파일 시스템, 입출력 관리, 네트워크 관리 등 모든 기능이 커널 내부에 포함된 단일형 커널
- 장점:
    - 시스템 호출 서비스가 빠름
    - 모든 기능이 커널 내부에 있어 효율성이 높음
- 단점:
    - 새로운 하드웨어 플랫폼에 대한 이식성이 떨어짐
    - 한 부분에서 발생한 문제가 시스템 전체에 영향을 줄 수 있음
    - 구성 요소 간 의존성이 높아 디버깅이 어려움

## 2. 마이크로커널 (Microkernel)

- 구조: 핵심 기능(프로세스 간 통신, 메모리 관리, 시계 인터럽트 처리, CPU 스케줄링 등)만을 커널에 포함하고, 나머지 기능(디바이스 드라이버 수행, 파일 시스템 등)은 서버 형태로 두어 사용자 모드의 프로세스로 수행
- 장점:
    - 유연성이 높음
    - 한 부분에서 발생한 오류가 시스템 전체로 확산되지 않음
    - 소형의 내장형 시스템에 활용하기에 유리하며, 원하는 기능만을 골라서 시스템 구축 가능
- 단점:
    - 파일 서비스 등에서 프로세스의 문맥 교환과 메시지 전송이 빈번해져 속도가 느려질 수 있음
