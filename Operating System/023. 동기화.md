# 동기화

---

## 1. 동기화(Synchronization)

- 여러 프로세스나 스레드가 동시에 실행될 때, 실행 순서를 제어하여 데이터 일관성을 보장하는 기술
- 주요 목적은 공유 자원에 대한 접근을 관리하여 데이터 무결성을 유지하고, 경쟁 조건과 교착 상태를 방지하는 것

## 핵심 개념

1. 데이터 무결성(Data Integrity): 데이터의 정확성, 일관성, 유효성이 유지되는 것.
2. 경쟁 조건(Race Condition): 여러 스레드가 동시에 공유 자원에 접근할 때, 실행 순서에 따라 결과가 달라질 수 있는 상황.
3. 데이터 일관성(Data Consistency): 여러 스레드가 공유 데이터를 일관되게 읽고 쓸 수 있도록 보장.
4. 교착 상태(Deadlock): 두 개 이상의 스레드가 서로가 가지고 있는 자원을 기다리느라 무한 대기 상태에 빠지는 상황.

### 동기화의 목적

- 실행 순서 제어(Execution Order Control): 작업이 올바른 순서로 실행되도록 한다.
- 상호 배제(Mutual Exclusion): 동시에 접근해서는 안 되는 자원에 하나의 작업만 접근하도록 한다.
- 동기화는 공유 자원에 대한 동시 접근으로 인한 데이터 일관성 문제를 해결하는 데 중요한 역할을 한다.
