# 교착상태 vs 기아 상태

---

## 1. 교착상태 (Deadlock)

- 교착상태는 둘 이상의 프로세스가 서로 자원을 기다리며 무한 대기 상태에 빠지는 현상
- 이러한 상태에서는 프로세스들이 더 이상 진행되지 않으며, 시스템 자원도 낭비
- 교착상태를 방지하거나 해결하기 위해서는 다음과 같은 방법들이 있다. 
  - 자원 할당 그래프(Resource Allocation Graph, RAG): 자원 할당 상황을 그래프로 나타내어 교착상태를 탐지합니다. 
  - 은행가 알고리즘(Banker's Algorithm): 시스템이 안전 상태를 유지하도록 자원 할당을 관리합니다. 
  - 교착상태 회피(Deadlock Avoidance): 자원 할당 시 교착상태 발생 가능성을 사전에 차단합니다. 
  - 교착상태 탐지 및 복구(Deadlock Detection and Recovery): 교착상태가 발생한 후 이를 탐지하고 복구합니다.

---

## 2. 기아상태 (Livelock)

- 기아상태는 프로세스들이 계속해서 상태를 변경하며 실행되지만, 실제로는 작업을 완료하지 못하는 상태
- 이는 교착상태와 유사하지만, 프로세스들이 계속해서 동작을 수행하는 점에서 차이가 있다. 
- 기아상태를 방지하기 위해서는 다음과 같은 방법들이 있다. 
  - 우선순위 조정(Priority Adjustment): 낮은 우선순위를 가진 프로세스에게 우선순위를 높여주어 기아상태를 방지합니다. 
  - 공정한 자원 배분(Fair Resource Allocation): 모든 프로세스가 자원을 공정하게 배분받도록 합니다. 
  - 재시도 메커니즘(Retry Mechanism): 기아상태에 빠진 프로세스가 일정 시간 후 다시 시도하도록 합니다.

## 3. 비교

| 특징                  | 교착상태 (Deadlock)                                           | 기아상태 (Livelock)                                         |
|-----------------------|--------------------------------------------------------------|------------------------------------------------------------|
| 정의                  | 두 개 이상의 프로세스가 서로 자원을 기다리며 무한 대기 상태에 빠짐 | 프로세스들이 계속해서 상태를 변경하지만 작업을 완료하지 못함   |
| 상태                  | 프로세스들이 멈추어 있음                                       | 프로세스들이 계속해서 실행됨                                |
| 원인                  | 자원 할당의 불완전한 상태                                      | 프로세스들이 자원을 얻으려 노력하지만 끝내 얻지 못하는 상황     |
| 해결 방법             | - 자원 할당 그래프 (RAG) 사용<br>- 은행가 알고리즘<br>- 교착상태 회피 | - 우선순위 조정<br>- 공정한 자원 배분<br>- 재시도 메커니즘       |
| 주요 특징             | 자원 낭비가 발생하며 시스템 정지 상태에 도달                   | 자원을 계속 요구하며 무한히 동작하지만 실제로는 진행되지 않음   |
| 상호작용              | 프로세스들이 서로 자원을 기다림                              | 프로세스들이 서로 상태를 변경하며 동작하지만 아무것도 완료되지 않음 |
