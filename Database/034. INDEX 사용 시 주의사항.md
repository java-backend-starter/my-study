# INDEX 사용시 주의사항

---

## 1. 인덱스 사용 시 유의사항

1. 카디널리티가 높은 열에 인덱스를 설정해야 한다.
    - 카디널리티는 특정 열의 데이터 값의 고유성을 나타낸다. 
    - 중복도가 높은 컬럼에 인덱스를 설정하면 성능 향상이 미미하므로 중복도가 낮은 열에 인덱스를 설정하는 것이 좋다.
2. WHERE 절에 자주 사용되는 열에 인덱스를 만들어야 한다.
    - WHERE 절에서 자주 사용되는 열에 인덱스를 설정하면 성능을 최적화할 수 있다. 
    - 자주 사용되지 않는 열에 인덱스를 설정해도 실제로 활용되지 않기 때문에 다른 열에 인덱스를 설정하는 것이 좋다.
3. 잘 사용되지 않는 인덱스는 제거해야 한다.
    - 인덱스는 테이블의 약 10%를 차지하며, 데이터 변경 시 페이지 분할 작업으로 성능에 악영향을 미칠 수 있다. 
    - 잘 사용되지 않는 인덱스는 제거하는 것이 성능을 개선하는 데 유리하다.
4. INSERT, UPDATE, DELETE 등의 데이터 작업이 빈번한 테이블에서는 인덱스 사용을 고려해야 한다.
    - 데이터 변경 작업이 자주 발생하는 테이블에서는 페이지 분할 작업으로 조회 성능 외의 다른 작업이 느려질 수 있다. 
    - 이런 경우, 인덱스를 사용하지 않고 약간의 조회 속도 저하를 감수하는 방법도 고려할 수 있다.
