# 정규화

---

## 1. 정규화

- 데이터베이스 정규화는 데이터의 중복을 제거하고 이상 현상을 방지하여 데이터의 일관성과 정확성을 유지하기 위한 과정입
- 이 과정은 일반적으로 1차부터 5차까지의 단계로 나뉨

### 이상 현상

- 정규화를 거치지 않은 데이터에서 발생할 수 있는 현상 
- 삽입 이상 (Insertion Anomaly) : 데이터 삽입 시 의도와 다른 값들도 삽입 
- 삭제 이상 (Delete Anomaly) : 데이터 삭제 시 의도와 다른 값들도 연쇄 삭제 
- 갱신 이상 (Update Anomaly) : 속성값 갱신 시 일부 튜플만 갱신되어 모순 발생

### 1차 정규화 (1NF)

- 각 컬럼의 값이 원자적이어야 합니다.
- 하나의 셀에는 하나의 값만 포함되어야 합니다.

### 2차 정규화 (2NF)

- 모든 비프라이머리 속성이 기본 키에 완전 함수 종속되어야 합니다.
- 부분적 종속성을 제거하기 위해, 기본 키의 일부에만 종속된 컬럼을 별도의 테이블로 분리합니다.

### 3차 정규화 (3NF)

- 비프라이머리 속성 간의 이행적 종속성을 제거합니다.
- 기본 키가 아닌 컬럼이 다른 비프라이머리 컬럼에 종속되지 않도록 합니다.

### 보이스-코드 정규형 (BCNF)

- 모든 결정자가 후보 키여야 합니다.
- 후보 키가 아닌 결정자가 존재하지 않도록 테이블을 분해합니다.

### 4차 정규화 (4NF)

- 다치 종속성을 제거합니다.
- 특정 속성 값에 따라 여러 값이 종속되는 경우, 이를 분리하여 독립적인 테이블로 만듭니다.

### 5차 정규화 (5NF)

- 조인 종속성을 제거합니다.
- 테이블을 분해하여도 자연 조인을 통해 원래의 테이블을 복원할 수 있도록 합니다.

각 단계는 이전 단계를 만족한 상태에서 수행되며, 데이터베이스의 무결성과 효율성을 높이는 데 기여합니다.
