# 일관성과 격리성

---

## 1. ACID란?

- 트랜잭션이 정상적으로 안전하게 수행되었는지를 보장하는 4가지 특성


### 원자성 (Atomicity)

- 트랜잭션 내 모든 작업이 전부 성공하거나 전부 실패해야 함
- 중간 실패가 발생하면 전체 트랜잭션이 무효화(Rollback)됨
- 예: 출금은 되었는데 입금은 실패한 상황 → 전체 작업 취소


### 일관성 (Consistency)

- 트랜잭션 실행 전과 후의 데이터는 항상 일관된 상태여야 함
- 데이터 무결성 제약 조건(예: 제약조건, 외래키 등)을 만족해야 함 
- 예: 입금 금액이 음수가 되거나, 없는 계좌로 입금되면 안 됨


### 격리성 (Isolation)

- 동시에 여러 트랜잭션이 수행될 때 서로 간섭 없이 독립적으로 실행되어야 함
- 트랜잭션의 중간 결과를 다른 트랜잭션이 볼 수 없음 
- 예: 한 사용자가 입금 중일 때, 다른 사용자는 그 중간 상태를 볼 수 없음


### 지속성 (Durability)

- 트랜잭션이 성공적으로 커밋되면, 그 결과는 영구적으로 저장됨
- 시스템 장애가 발생해도 해당 데이터는 손실되지 않음 
- 예: 전원이 꺼져도 커밋된 데이터는 복구 가능해야 함

---

## 2. 요약표

| 특성             | 설명 |
|------------------|------|
| 원자성           | 트랜잭션은 전부 성공하거나 전부 실패 |
| 일관성           | 데이터는 항상 무결성과 일관성을 유지 |
| 격리성           | 트랜잭션 간 서로 간섭하지 않도록 격리 |
| 지속성           | 성공한 트랜잭션 결과는 영구적으로 저장 |
