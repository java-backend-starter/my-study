# SQL vs NoSQL

---

## 1. SQL (Structured Query Language)

### 개요
- Structured Query Language의 줄임말
- 관계형 데이터베이스에 정보를 저장하고 처리하기 위한 프로그래밍 언어
- 관계형 데이터베이스: 데이터를 행과 열의 2차원 표 형태로 저장하는 데이터베이스

### 구조와 스키마
- 관계형 데이터베이스(RDBMS)로, 고정된 스키마를 기반으로 데이터를 테이블 형식으로 저장
- 각 테이블은 명확한 구조를 가지며, 데이터 무결성을 보장

### 확장성
- 주로 수직적 확장(하드웨어 업그레이드)을 통해 성능 향상

### 장단점
- 장점
    - 명확한 스키마로 데이터 무결성 보장
    - 복잡한 쿼리와 조인 연산 지원
- 단점
    - 스키마 변경이 어렵고 유연성이 낮음
    - 수평적 확장이 어려움

---

## 2. NoSQL (Not Only SQL)

### 개요
- 초고용량 데이터 처리 등 성능에 특화된 목적을 위해 개발된 비관계형 데이터 저장소
- "Not Only SQL"은 기존 관계형 데이터베이스의 특성 외에도 다양한 저장 방식을 지원한다는 의미를 가짐
- 비관계형 데이터베이스(NoSQL)도 관계형 데이터 저장이 가능한 경우가 있음 (예: 그래프 데이터베이스)
- 일반적으로 관계형 DB의 주요 특성인 ACID(Atomicity, Consistency, Isolation, Durability)를 완전히 보장하지 않음

### 구조와 스키마
- 비관계형 데이터베이스로, 스키마 없이 다양한 형태의 데이터를 저장 가능
- 문서(Document), 키-값(Key-Value), 컬럼 패밀리(Column-Family), 그래프(Graph) 등 다양한 데이터 모델 지원

### 확장성
- 수평적 확장(서버를 추가하여 성능 향상)이 가능

### 장단점
- 장점
    - 유연한 스키마로 데이터 구조 변경 용이
    - 대량의 비정형 데이터 처리에 적합
    - 수평적 확장을 통한 높은 확장성
- 단점
    - 데이터 중복이 발생할 수 있으며, 중복된 데이터의 일관성 유지가 어려움
    - 복잡한 쿼리나 조인 연산이 제한적
    - 일부 NoSQL 데이터베이스는 ACID 트랜잭션 지원이 제한적

---

## 3. 선택 기준

- SQL 데이터베이스
    - 명확한 스키마와 데이터 무결성이 중요한 시스템에 적합
- NoSQL 데이터베이스
    - 데이터 구조가 자주 변경되거나 대량의 비정형 데이터를 처리해야 하는 경우
    - 수평적 확장이 필요한 시스템에 적합
