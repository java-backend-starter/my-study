# 비정규화

---


## 1. 역정규화(denormalization)

- 데이터베이스에서 성능 향상이나 개발 편의성을 위해 정규화를 통해 분리된 테이블을 다시 통합하거나 구조를 변경하는 과정입니다. 정규화는 데이터의 중복을 최소화하고 무결성을 유지
- 과도한 정규화는 읽기 성능 저하를 초래할 수 있음
- 역정규화를 통해 읽기 성능을 개선할 수 있음

---

## 2. 주요 특징

- 읽기 성능 향상: 테이블을 통합하여 조인(join) 연산을 줄임으로써, 데이터 조회 시 성능을 높일 수 있음.
- 데이터 중복 허용: 성능 개선을 위해 일부 데이터 중복을 허용하며, 이는 저장 공간의 증가를 초래할 수 있음

---

## 3. 역정규화의 장단점:

### 장점

- 쿼리 성능 개선: 조인 연산 감소로 인해 읽기 성능이 향상
- 쿼리 복잡도 감소: 단순한 쿼리로 원하는 데이터를 쉽게 조회할 수 있음

### 단점

- 데이터 일관성 문제: 중복된 데이터로 인해 삽입, 수정, 삭제 시 데이터 불일치가 발생할 수 있음
- 저장 공간 증가: 중복 데이터 저장으로 인해 저장 공간이 늘어남

---

### 4. 역정규화 적용 시 고려사항

- 정규화 선행: 역정규화를 적용하기 전에 반드시 정규화를 수행하여 데이터 구조를 최적화
- 신중한 적용: 역정규화는 시스템의 복잡도를 증가시키므로, 성능 개선이 필요한 부분에 한정하여 신중하게 적용
- 따라서, 역정규화는 데이터베이스의 성능 요구 사항과 데이터 무결성 유지 사이의 균형을 고려하여 적용

---

### 5. 역정규화의 종류

- 컬럼 조작: 조인을 줄이기 위해 관련 데이터를 같은 테이블에 중복 저장
- 파생 컬럼 추가: 집계 계산을 줄이기 위해 계산 결과를 컬럼으로 저장
- 테이블 분할: 컬럼 또는 행 기준으로 분리하여 성능 개선
- 관계 역정규화: 조인을 줄이기 위해 외래키를 추가로 삽입
