# 제약조건

---

## 1. 제약조건이란?

- 테이블에 저장되는 데이터가 사용자가 정의한 조건을 만족하도록 강제하는 규칙으로, 데이터의 무결성을 보장하는 역할
- 잘못된 데이터가 테이블에 입력되지 않도록 제한
- 제약조건은 데이터 입력 시 자동으로 검사되며, 조건을 만족하지 않으면 입력/수정/삭제가 거부
- 제약조건은 다음 두 가지 방법으로 설정할 수 있음:
    - 테이블 생성 시: `CREATE TABLE` 문
    - 기존 테이블 수정 시: `ALTER TABLE` 문

---

## 2. Oracle SQL Developer에서 주로 사용되는 제약조건

| 제약조건       | 설명 |
|----------------|------|
| `NOT NULL`     | 컬럼에 `NULL` 값을 허용하지 않음. 반드시 값이 있어야 함. |
| `UNIQUE`       | 중복된 값을 허용하지 않음. 컬럼의 값이 고유해야 함. |
| `PRIMARY KEY`  | 행(row)을 고유하게 식별하는 키. `NOT NULL` + `UNIQUE`의 성격을 모두 가짐. |
| `FOREIGN KEY`  | 다른 테이블의 `PRIMARY KEY`를 참조하여 테이블 간의 관계를 유지함. |
| `CHECK`        | 컬럼에 입력될 수 있는 값을 조건으로 제한함. 예: `CHECK (age > 0)` |
| `DEFAULT`      | 명시된 값이 없을 경우 자동으로 기본값을 입력함. |
| `AUTO_INCREMENT` (MySQL 등) | 자동으로 증가하는 숫자 값을 부여함. Oracle에서는 `SEQUENCE` + `TRIGGER`로 구현함. |
- 제약조건은 데이터 무결성을 보장하기 위한 필수 도구
- 상황에 따라 조합해서 사용해야 데이터의 신뢰성과 안정성을 확보할 수 있음
