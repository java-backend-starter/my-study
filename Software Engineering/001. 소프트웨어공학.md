# 소프트웨어 공학

---

## 1. 소프트웨어 공학 (Software Engineering)

- 소프트웨어 위기를 극복하고 품질 높은 소프트웨어를 효율적으로 개발하기 위한 학문으로, 개발 방법, 도구, 이론 등을 포괄

---


## 2. 소프트웨어 위기

- 급격한 하드웨어 기술의 발전과 그에 따른 사용자 요구사항의 다양화에 비해, 소프트웨어 기술의 발전 속도가 더딘 현상을 말한다. 
- 이로 인해 소프트웨어 개발과 유지보수에 다양한 문제가 발생하게 된다.

### 원인

- 소프트웨어 특성에 대한 이해 부족
- 적절한 관리 방법론의 부재
- 설계의 소홀함
- 전문 교육의 부족
- 일정 및 비용 추정의 부정확성

### 구체적인 위기 현상

- 개발 일정이 지연된다.
  - 초기 계획보다 프로젝트가 늦어지는 문제가 발생한다.
- 초과 비용이 발생한다. 
  - 예산을 초과하는 개발 비용이 필요해진다. 
- 제품의 신뢰도가 결여된다. 
  - 시스템의 안정성이 낮고 오류가 자주 발생한다. 
- 명세와 불일치하는 부분이 자주 나타난다.
  - 요구사항과 실제 구현 결과가 다르게 되는 경우가 많다.
- 품질이 저하되고 유지보수가 어렵다. 
  - 코드 품질이 낮아지고 수정이나 기능 추가가 어려워진다.
---

## 3. 소프트웨어 공학의 3R

- 역공학 (Reverse Engineering): 기존 시스템을 분석하여 요구사항이나 설계 문서를 추출하는 작업.
- 재공학 (Re-Engineering): 기존 시스템을 분석하고 새로운 형태나 구조로 변형하는 과정.
- 재사용 (Reuse Engineering): 기존에 개발된 소프트웨어를 재활용하여 새로운 소프트웨어를 개발하는 과정.

---

## 4. 소프트웨어 개발 단계

1. 계획 (Planning): 프로젝트 목표, 범위, 일정, 비용 등을 설정.
2. 요구사항 분석 (Requirements Analysis): 사용자 및 시스템 요구사항 수집, 분석, 문서화.
3. 설계 (Design): 시스템 아키텍처 및 세부 구성 요소 설계.
4. 구현 (Implementation): 설계에 따라 소프트웨어 코드 작성 및 개발.
5. 테스트 (Test): 개발된 소프트웨어의 기능 및 성능 검증.
6. 유지보수 (Maintenance): 운영 중 발생하는 버그 수정, 기능 추가, 성능 향상 작업.
