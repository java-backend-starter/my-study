# UML

---

## 1. UML 다이어그램이란?

- UML (Unified Modeling Language)
    - 시스템 구조와 기능을 시각화하고 문서화하는 표준화된 모델링 언어
- 사용 목적
    - 소프트웨어 및 비즈니스 모델링
    - 시스템 설명, 설계 검증, 문서화
- 특징
    - 그래픽 표기법 사용
    - 객체지향 소프트웨어 개발에 효과적
    - 대규모 시스템 모델링에 적합
    - 팀 간 의사소통 도구로 활용

---

## 2. UML 사용 이유

- 소프트웨어의 전략적 가치 증가 → 자동화, 품질 개선, 비용 절감 필요
- 해결해야 할 주요 아키텍처 문제:
    - 물리적 분산, 동시성, 보안, 복제, 로드밸런싱, 내결함성 등
- Page-Jones의 UML 설계 목표:
    - 즉시 사용 가능한 시각적 모델링 언어 제공
    - 확장성 및 전문화 지원
    - 프로그래밍 언어 및 개발 프로세스 독립적
    - 공식적인 기반 제공
    - 상위 개념(패턴, 프레임워크, 협업 등) 지원
    - 모범 사례 통합

---

## 3. UML 이해관계자

- 다양한 역할 → 다양한 시각 요구됨
- 주요 이해관계자:
    - 분석가 (Analyst)
    - 디자이너 (Designer)
    - 개발자 (Coder)
    - 테스터 (Tester)
    - 고객 (Customer)
    - QA 테스터
    - 기술 문서 작성자 (Technical Author)
- UML의 역할
    - 각 이해관계자가 자신의 관점에서 시스템을 시각화할 수 있도록 지원

---

## 4. UML 다이어그램 분류 (UML 2.x 기준)

### 구조 다이어그램 (Structure Diagrams)

- 시스템의 정적 구조 표현
- 종류:
    - Class Diagram
    - Component Diagram
    - Deployment Diagram
    - Object Diagram
    - Package Diagram
    - Composite Structure Diagram
    - Profile Diagram

### 행위 다이어그램 (Behavior Diagrams)

- 시간에 따른 동작 및 상호작용 표현
- 종류:
    - Use Case Diagram
    - Activity Diagram
    - State Machine Diagram
    - Sequence Diagram
    - Communication Diagram
    - Interaction Overview Diagram
    - Timing Diagram
