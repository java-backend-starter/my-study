# 인증(Authentication)과 인가(Authorization)의 차이

---

## 1. 인증(Authentication)

- 정의: 사용자의 신원을 확인하는 과정으로, 보안 프로세스의 첫 번째 단계입니다. 
- 목적: 사용자가 누구인지 확인합니다. 
- 주요 방법
  - 비밀번호: 사용자 이름과 비밀번호를 입력하여 신원을 확인합니다. 
  - 일회용 PIN: 단일 세션에 대한 액세스를 허용하는 일회용 코드를 사용합니다. 
  - 인증 앱: 보안 코드를 생성하는 외부 기관을 통해 확인합니다. 
  - 생체인식: 지문이나 망막 스캔을 통해 인증합니다.
- 특징: 인증 후 사용자에게 시스템 접근을 허용합니다. 다중 요소 인증(MFA)을 사용해 보안을 강화할 수 있습니다.

---

## 2. 인가(Authorization)

- 정의: 인증된 사용자가 특정 리소스나 기능에 액세스할 수 있는 권한을 부여하는 과정입니다. 
- 목적: 사용자가 어떤 작업을 할 수 있는지, 어떤 리소스에 접근할 수 있는지 결정합니다. 
- 예시
  - 서버에서 특정 파일을 다운로드할 수 있는 권한을 부여합니다. 
  - 개별 사용자에게 관리자 권한을 부여합니다.
- 특징: 인증 후에 진행되며, 권한 부여는 보안 팀의 설정에 의해 관리됩니다.

---

## 3. 인증과 인가의 차이


- 인증과 인가는 로그인 프로세스에서 각각 다른 단계를 담당하며, IAM(Identity and Access Management) 솔루션을 성공적으로 구현하려면 이 둘의 차이를 잘 이해해야 한다.

### 비유를 통한 설명

- 인증
  - 사용자가 자격 증명을 통해 신원을 확인하는 과정이다. 
  - 예를 들어, 집에 들어가기 위해서는 현관 자물쇠에 맞는 열쇠가 필요하다. 
  - 이처럼 인증은 사용자가 올바른 자격 증명을 입력할 때만 시스템에 접근할 수 있도록 허용한다.
- 인가
  - 집에 들어간 후, 특정 공간에 접근할 수 있는 권한을 부여받는 과정이다.
  - 예를 들어, 주방에 가서 반려 동물 사료를 보관한 찻장을 열 수 있는 권한을 얻지만, 침실에 들어갈 권한은 없을 수 있다. 
  - 이는 인증 후 특정 리소스에 대한 접근 권한을 부여받는 권한 인증이다.

### 인증과 인가의 차이점

| 기능               | 인증 (Authentication)                           | 인가 (Authorization)                       |
|-------------------|------------------------------------------------|------------------------------------------|
| 기능               | 자격 증명 확인                                  | 권한 허가/거부                           |
| 진행 방식         | 비밀번호, 생체인식, 일회용 PIN 또는 앱 사용     | 보안 팀에서 관리하는 설정 사용           |
| 사용자가 볼 수 있는가? | 예                                               | 아니오                                    |
| 사용자가 직접 변경할 수 있는가? | 부분적으로 가능                                    | 불가능                                   |
| 데이터 전송        | ID 토큰 사용                                    | 액세스 토큰 사용                          |

### 시스템 적용

- 인증: 사용자가 올바른 자격 증명을 입력하면 기업 시스템에 대한 액세스를 허용한다.
- 인가: 인증된 사용자에게 부서별 파일 접근 권한을 부여하고, 필요할 경우 기밀 데이터에 대한 접근 권한도 부여한다.

### 결론

- 인증과 인가는 서로 다른 보안 프로세스를 담당하며, IAM 관리자는 이 두 가지 프로세스의 활용 방식을 잘 이해하고 시스템에 적절히 적용해야 한다.
