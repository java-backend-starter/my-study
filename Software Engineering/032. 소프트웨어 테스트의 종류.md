# 소프트웨어 테스트의 종류

---

## 1. 테스트 분류 기준

- 테스트는 다음 세 가지 기준으로 분류된다
1. 테스트 레벨  
   - 개발 단계에 따른 테스트 범위
2. 테스트 유형  
   - 어떤 목적의 테스트인가?
3. 테스트 설계 기법  
   - 테스트 케이스를 어떻게 도출할 것인가?
- 이 세 가지 분류 기준은 서로 독립적이며, 다양한 조합으로 사용된다.

---

## 2. 테스트 레벨에 따른 분류

### 컴포넌트(단위) 테스트

- 단일 모듈 또는 유닛을 대상으로 하는 테스트
- 내부 로직이나 인터페이스가 예상대로 동작하는지 확인
- 일반적으로 개발자가 작성하며, 테스트 도구 사용 가능 (예: JUnit)

### 통합 테스트

- 여러 모듈을 조합하여 테스트
- 모듈 간 인터페이스 및 상호작용이 올바른지 검증
- 하향식 / 상향식 / 빅뱅 통합 방식 등이 있음

### 시스템 테스트

- 전체 시스템을 하나의 단위로 보고 수행
- 기능, 성능, 보안 등 시스템 요구사항 충족 여부를 확인

### 인수 테스트

- 최종 사용자의 요구사항 충족 여부 확인
- 보통 고객, 사용자, QA팀이 수행
- 운영 환경과 유사한 환경에서 테스트 진행

### V 모델

- 테스트 레벨과 개발 단계의 관계를 시각화한 모델
- 왼쪽: 개발 단계 (요구사항 → 설계 → 구현)
- 오른쪽: 대응되는 테스트 단계 (인수 → 시스템 → 통합 → 단위)

---

## 3. 테스트 유형에 따른 분류

### 기능 테스트

- 소프트웨어가 기능적 요구사항을 충족하는지 검증
- 예: 로그인 기능, 계산기 기능 등 사용자의 기대 동작

### 비기능 테스트

- 소프트웨어의 품질 특성을 검증 (ISO/IEC 25010 기준)
- 시스템의 성능, 보안, 사용성, 이식성 등 확인

#### 비기능 품질 특성과 세부 특성

| 주 특성         | 세부 특성 예시                            |
|------------------|---------------------------------------------|
| 기능 적합성       | 완전성, 정확성, 타당성                         |
| 성능 효율성       | 응답 시간, 처리량, 자원 활용성                 |
| 호환성           | 다양한 환경/시스템 간 공존성, 상호운용성       |
| 사용성           | 학습성, 사용 용이성, UI의 직관성 등             |
| 신뢰성           | 장애 발생 시 대처 능력, 오류 발생률             |
| 보안성           | 기밀성, 무결성, 인증, 권한 관리                |
| 유지보수성       | 모듈화, 분석성, 테스트 용이성, 수정 용이성       |
| 이식성           | 다양한 플랫폼에서의 적응성, 설치 용이성         |

## 4. 테스트 설계 기법에 따른 분류

- 테스트 케이스를 어떻게 도출하는가에 따라 분류함

### 정적 테스트 (코드 실행 없이)

- 문서, 설계서, 코드 등을 실행하지 않고 검토

#### 리뷰

- 산출물의 결함 탐지 및 품질 향상 목적
- 종류:
    - 관리 리뷰: 프로젝트 상태 검토
    - 기술 리뷰: 설계/요구사항 문서 검토
    - 워크쓰루: 비공식 리뷰
    - 인스펙션: 공식적 절차, 결함 탐지 중심
    - 감사: 규정 및 표준 준수 확인

#### 정적 분석

- 자동화 도구를 활용하여 코드 품질 점검
- 검사 항목:
    - 코딩 표준 준수 (예: MISRA)
    - 복잡도 측정 (사이클로매틱 복잡도)
    - 데이터 흐름, 변수 사용 등

### 동적 테스트 (코드 실행 기반)

- 프로그램을 실행하면서 동작을 검증

#### 명세 기반 기법

- 요구사항 명세서를 바탕으로 설계
- 예:
    - 동등 분할
    - 경계값 분석
    - 상태 전이 테스트
    - 결정 테이블
    - 유스케이스 기반 테스트

#### 구조 기반 기법

- 코드의 구조나 흐름을 기준으로 테스트
- 예:
    - 문장 커버리지
    - 분기 커버리지
    - 조건/결정 커버리지
    - 경로 커버리지

#### 경험 기반 기법

- 테스터의 경험이나 직관을 바탕으로 수행
- 예:
    - 탐색적 테스트
    - 오류 추정
    - 체크리스트 기반 테스트

## 4. 테스팅 방법

### 리그레션 테스트 (회귀 테스트)

- 코드 수정 후 기존 기능이 의도대로 동작하는지 재검증
- 유지보수 시 필수 수행
- 자동화 테스트 활용도가 높음

#### 리그레션 테스트 전략

1. Retest All: 전체 테스트 케이스 다시 수행 (비용↑)
2. 선택적 테스트: 영향받은 부분만 테스트
3. 테스트 최소화: 테스트 케이스 수 줄이기
4. 테스트 우선순위화: 위험도나 빈도 기준으로 우선 수행

### 소프트웨어 생명주기 모델과 테스트

- 테스트는 각 개발 단계와 연계되어 계획되어야 함
- 개발 모델 예시:
    - 폭포수 모델: 개발 완료 후 테스트 진행
    - V-모델: 개발 단계마다 대응되는 테스트 있음
    - 애자일 모델: 반복적/점진적 개발과 짝지어진 테스트

### 위험 기반 테스트

- 정의: 테스트 대상의 자원을 활용하여 위험 분석을 바탕으로 테스트 범위와 접근 방법을 결정하는 기법
- 핵심 아이디어: 테스트 제외로 발생할 수 있는 위험 수준이 높은 부분은 반드시 테스트 대상에 포함시켜야 함
- 실행 방식: 기능(feature)에 대한 위험 분석을 바탕으로 테스트 계획, 설계, 실행 등의 활동을 수행

### 모델 기반 테스트

- 정의: 테스트 대상의 기대 동작을 표현하는 정형화되고 상세한 모델을 기반으로 테스트 절차를 수행하는 기법
- 핵심 아이디어: 모델 기반 테스트는 테스트 절차를 수행할 수 있는 정보가 자동으로 추출될 수 있을 정도로 정형화되고 상세한 모델을 바탕으로 함
- 예시:
    - 자연어로 된 표현 형식
    - 상태 전이도와 같은 시각적인 표현 방식
    - 의사결정표와 같은 표 형태
