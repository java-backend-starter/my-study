# XSS(Cross-Site Scripting)

---

## 1. XSS란?

- "Cross Site Scripting"의 약자. 
- 웹사이트에 악성 스크립트를 삽입해, 클라이언트(사용자 브라우저) 측에서 실행되게 하는 해킹 공격 기법.
- "CSS"와 헷갈리지 않기 위해 "XSS"라고 부름. 
- 매우 기초적이면서도 범위가 넓고 빈번하게 발생하는 취약점. 
- 공격자는 취약한 사이트에 스크립트 삽입 또는 페이로드 링크를 만들어 피해자가 접속하면, 브라우저가 이를 실행함. 
- 공격 대상: 일반 사용자뿐만 아니라 관리자 권한을 가진 사용자도 될 수 있음. 
- 주로 자바스크립트를 사용하지만, HTML, CSS 삽입도 포함됨.

---

## 2. XSS 공격 기법 종류

- Stored XSS (저장형)
  - 악성 스크립트가 서버에 저장되어 여러 사용자가 접속할 때마다 공격됨. 
- Reflected XSS (반사형)
  - 입력값이 반사되어 URL에 삽입, 피해자가 링크를 클릭하면 공격 발생. 
- DOM Based XSS
  - 클라이언트 측에서 DOM 조작을 통해 발생하는 XSS.

---

## 3. XSS 위험성

- 피해자가 URL을 클릭해야 하는 Reflected XSS는 제한적이지만, Stored XSS는 접속자 수에 따라 피해 범위가 대규모로 확장될 수 있음.
- 발생할 수 있는 피해 
  - 개인정보 노출 
  - 웹 사용 불편 
  - 쿠키 탈취 및 세션 하이재킹 
  - 악성코드 다운로드 
  - 피싱 공격 
  - CSRF 공격 유발 
  - 키로깅

---

## 4. 대응법

### 개발자 측 대응

- 입력값 검증 및 필터링 
- 특수문자 HTML Entity 치환 
- innerHTML 사용 지양 
- 쿠키에 httpOnly 옵션 설정 
- Content Security Policy(CSP) 적용 
- 인증 강화 
- 쿠키에 민감한 정보 저장 금지 
- 보안 검증된 API 사용

### 사용자 측 대응

- 주기적인 비밀번호 변경 
- 브라우저를 최신 버전으로 유지 
- 수상한 링크 접근 금지 
- 브라우저 보안 설정 강화
