# 회귀 테스트

---

## 1. 회귀 테스트란?

- 회귀 테스트(Regression Test)는 소프트웨어에 새로운 기능을 추가하거나 기존 기능을 수정한 후, 이러한 변경이 기존 기능에 부정적인 영향을 미치지 않았는지 확인하는 테스트이다. 
- 즉, 이전에 정상적으로 작동하던 기능이 변경으로 인해 문제가 생기지 않았는지를 검증한다. 
- 이를 위해 이전에 실행한 테스트를 다시 수행하여 기존 기능에 문제가 없는지 확인한다.

---

## 2. 회귀 테스트 케이스 관리

- 회귀 테스트 케이스는 일반적으로 다음과 같은 방식으로 관리된다:
  1. 이번 버전에서 수행한 테스트 케이스를 다시 실행하여 변경된 부분이 기존 기능에 영향을 미치지 않았는지 확인한다.
  2. 기존에 관리하고 있던 회귀 테스트 케이스를 별도로 수행하여 전체적인 기능의 안정성을 검증한다.
- 회귀 테스트 케이스는 팀 내부 방침에 따라 작성되며, 
- 보통 기존 기능들의 일반적인 패턴을 추려 작성하고, 매 버전마다 추가 및 관리하여 사용한다.

---

## 3. 회귀 테스트 유형
- Retest All
  - 기존의 모든 테스트 케이스를 다시 실행한다. 
  - 테스트 커버리지는 높지만, 시간과 비용이 많이 소요된다.
- Selective 
  - 변경된 부분과 관련된 테스트 케이스만 선택하여 실행한다. 
  - 시간과 비용을 절약할 수 있지만, 테스트 커버리지가 낮아 결함을 놓칠 수 있다.
- Priority 
  - 시스템의 핵심 기능을 우선순위로 정하여 테스트를 수행한다. 
  - 중요한 기능에 집중할 수 있지만, 우선순위 선정이 잘못되면 결함을 발견하지 못할 수 있다.

---

## 4. 자동화 회귀 테스트

- 회귀 테스트는 반복적으로 수행되기 때문에 자동화의 대상이 된다.
- 자동화 방법은 다음과 같다:
  - 3rd Party 제품을 활용한 Recording & Play
    - 테스트 과정을 녹화하여 재생하는 방식이다.
  - 직접적인 코딩을 통한 자동 회귀 테스트 구축
    - 코드를 작성하여 테스트를 자동화하는 방식이다.
- 그러나 자동화 회귀 테스트는 유지보수 비용이 발생하며, 특히 UI가 자주 변경되는 경우에는 적합하지 않을 수 있다. 
- 따라서 테스트 시나리오와 테스트 케이스 설계가 잘 이루어져야 자동화의 효과를 높일 수 있다.
