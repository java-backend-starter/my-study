# DRY 원칙

---

## 1. DRY (Don't Repeat Yourself)

- 정의: "자신을 반복하지 말라"는 의미로, 동일한 코드나 로직을 여러 곳에 중복해서 작성하지 말라는 원칙이다.
- 목적: 중복을 제거하여 유지보수성을 높이고, 코드의 일관성을 유지하며, 버그 발생 가능성을 줄인다.
- 적용 예시: 공통 기능을 하나의 함수나 모듈로 분리하여 여러 곳에서 재사용하는 방식.

---

## 2. 원칙 비교

| 원칙  | 중점 사항           | 복잡성 관리 방식                 | 적용 시기            |
|-------|---------------------|----------------------------------|----------------------|
| DRY   | 코드 중복 제거       | 중복 코드를 모듈화하여 재사용     | 구현 단계            |
| KISS  | 설계의 단순성 유지   | 불필요한 복잡성 제거             | 설계 및 구현 단계     |
| YAGNI | 불필요한 기능 배제   | 현재 필요한 기능만 구현          | 요구사항 분석 및 구현 단계 |

---

## 3. 관계 및 통합적 관점

- DRY는 코드의 중복을 제거하여 유지보수를 용이하게 한다.
- KISS는 시스템의 복잡성을 줄여 이해하기 쉬운 구조를 만든다.
- YAGNI는 불필요한 기능 구현을 방지하여 개발 효율성을 높인다.
