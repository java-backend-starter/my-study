# MVVM 패턴이란?

- MVVM은 Model, View, ViewModel의 약자로, 비즈니스 로직과 프레젠테이션 로직을 UI로부터 명확하게 분리하는 아키텍처 패턴
- 코드의 유지 보수성과 테스트 용이성을 높일 수 있습니다.

## 구성 요소

- Model: 데이터와 비즈니스 로직을 담당하며, 데이터베이스, 네트워크 요청 등과 상호작용
- View: 사용자 인터페이스를 담당하며, 사용자 입력을 처리하고 화면을 표시
- ViewModel: View와 Model 사이에서 중재자 역할을 하며, View의 이벤트를 처리하고 Model과 상호작용하여 데이터를 가공한 후 View에 제공

## 동작 과정

1. 사용자의 액션이 View를 통해 들어옵니다.
2. View는 해당 액션을 ViewModel에 전달합니다.
3. ViewModel은 필요한 데이터를 Model에 요청합니다.
4. Model은 요청받은 데이터를 ViewModel에 응답합니다.
5. ViewModel은 받은 데이터를 가공하여 저장합니다.
6. View는 데이터 바인딩을 통해 UI를 갱신합니다.

## 특징

- MVP 패턴과 유사하지만, MVVM에서는 View와 ViewModel 사이의 관계가 1:n으로 설정될 수 있다.
- 데이터 바인딩을 활용하여 View와 ViewModel 간의 의존성을 줄일 수 있다.

## 장점

- 뷰 로직과 비즈니스 로직의 분리를 통해 생산성을 향상시킬 수 있다.
- 의존성이 낮아 테스트가 용이하다.
- ViewModel의 재사용성을 통해 코드 중복을 줄일 수 있다.
- 많은 기업에서 채택하고 있는 디자인 패턴이다.

## 단점

- Rx나 데이터 바인딩에 대한 이해가 필요하여 설계가 복잡할 수 있다.
- ViewModel이 비대해질 수 있다.
- 데이터 바인딩으로 인한 메모리 소모가 증가할 수 있다.
