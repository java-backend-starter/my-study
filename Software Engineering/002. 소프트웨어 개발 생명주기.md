# 소프트웨어 개발 생명주기(SDLC)

---

## 0. SDLC란?

- 소프트웨어를 계획 → 개발 → 테스트 → 배포 → 유지보수하는 전 과정을 구조화한 개발 절차
- 소프트웨어 개발에 대한 단계별 접근 방식
- 시스템의 요구 분석부터 유지보수까지 전 과정을 체계화한 절차
- 개발 프로세스를 단계별로 단순화하고 추상화한 것

---

## 1. 주먹구구식 모델 (Code-and-Fix Model)

- 특징: 계획 없이 바로 코딩 → 오류 수정 반복
- 장점:
    - 빠르게 개발 시작 가능
    - 소규모 프로젝트에 적합
- 단점:
    - 유지보수 어려움
    - 협업 및 확장성 부족
    - 코드 가독성 저하

---

## 2. 폭포수 모델 (Waterfall Model)

- 특징: 단계별 순차적 진행 (계획 → 분석 → 설계 → 구현 → 테스트)
- 장점:
    - 절차가 명확하고 이해하기 쉬움
    - 단계별 진척사항 관리 용이
    - 체계적인 문서화 가능
- 단점:
    - 요구사항 변경에 취약
    - 중간 결과물 확인 불가
    - 원하는 결과가 아닐 경우 수정에 많은 시간과 비용 소요

---

## 3. 프로토타입 모델 (Prototyping Model)

- 특징: 시제품(프로토타입)을 제작하여 사용자 피드백을 반영
- 장점:
    - 고객 요구사항 정확히 파악 가능
    - 기능 구현 예측 용이
    - 사용자와의 원활한 커뮤니케이션 가능
- 단점:
    - 개발 범위 불명확
    - 인력 및 비용 산정 어려움
    - 고객과의 지속적인 협업 필요

---

## 4. 점증적 모델 (Incremental Model)

- 특징: 기능 단위로 점진적 개발 및 릴리즈
- 장점:
    - 초기 제품 빠르게 출시 가능
    - 사용자 피드백 반영 용이
- 단점:
    - 통합 시 복잡성 증가
    - 유지보수 어려움

---

## 5. 나선형 모델 (Spiral Model)

- 특징: 반복적 개발과 위험 분석 병행
- 장점:
    - 리스크 조기 식별 및 대응 가능
    - 대규모, 고위험 프로젝트에 적합
- 단점:
    - 구조 복잡, 전문적인 관리 필요
    - 높은 비용과 긴 개발 시간
    - 위험 분석에 대한 전문 지식 요구

---

## 6. 통합 프로세스 모델 (Unified Process, UP)

- 특징: 반복적이며 점진적인 개발 모델 (도입 → 구체화 → 구축 → 전이)
- 장점:
    - 반복적 접근으로 리스크 관리 용이
    - 복잡한 시스템에 적합
- 단점:
    - 관리 복잡
    - 학습 곡선 높음

---

## 7. V 모델 (Verification & Validation Model)

- 특징:
    - 개발 단계와 병행하여 테스트 진행
    - 단위 테스트 → 통합 테스트 → 시스템 테스트 → 인수 테스트 구조
- 장점:
    - 각 단계별 검증을 통해 높은 품질 확보
- 단점:
    - 유연성 부족
    - 초기부터 테스트 계획이 필요해 부담이 큼
- 개념 정리:
    - Verification: 개발자 관점에서의 검증
    - Validation: 사용자 관점에서의 확인

---

## 8. 애자일 방법론 (Agile Methods)

- 특징: 짧은 개발 주기와 반복을 기반으로 한 고객 중심 개발 방식
- 장점:
    - 변화에 유연한 대응
    - 빠른 피드백과 개선 가능
- 단점:
    - 문서화 부족
    - 대규모 프로젝트에서는 관리 어려움
- 주요 방식:
    - eXtreme Programming (XP)
    - Scrum
    - Lean Software Development
    - DevOps

| 모델 | 장점 | 단점 | 추천 상황 |
|------|------|------|------------|
| 주먹구구식 | 빠른 개발 시작 | 유지보수 불가 | 소규모 개인 프로젝트 |
| 폭포수 | 절차 명확, 문서화 | 변경 대응 불가 | 요구사항 고정된 프로젝트 |
| 프로토타입 | 요구 반영 쉬움 | 범위 모호 | 사용자 중심 개발 |
| 점증적 | 빠른 릴리즈 | 통합 복잡 | 민첩한 릴리즈 필요 시 |

| 통합 프로세스 | 반복적 개선 | 관리 어려움 | 복잡한 시스템 개발 |
| V 모델 | 품질 확보 | 유연성 부족 | 테스트 중시 시스템 |
| 애자일 | 유연, 빠른 대응 | 문서화 약함 | 빠른 피드백 필요 시 |
