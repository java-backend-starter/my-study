# 기술 부채

---

## 1. 개요

- 기술 부채(Technical Debt)는 소프트웨어 개발 과정에서 단기적인 이득이나 빠른 출시를 위해 최적이 아닌 설계나 구현을 선택함으로써, 향후 그로 인한 추가 작업이나 수정이 필요한 상태를 의
- 이는 마치 금융 부채와 같이 시간이 지남에 따라 '이자'가 붙어 미래의 개발 속도와 품질에 부정적인 영향을 미칠 수 있음

---

## 2. 기술 부채의 주요 원인

1. 개발자의 역량 부족: 경험이나 지식 부족으로 인해 비효율적인 코드가 작성되는 경우
2. 빠른 시장 출시 압박: 제품을 신속하게 출시하기 위해 임시방편을 선택하는 경우
3. 불완전한 설계: 초기 설계 단계에서 충분한 고려 없이 진행된 경우
4. 기술 변화 미흡: 최신 기술이나 표준을 따라가지 못해 기존 시스템이 노후화되는 경우

---

## 3. 기술 부채의 주요 유형

- 설계 부채: 부적절하거나 미흡한 소프트웨어 설계로 인한 부채
- 코드 부채: 비효율적이거나 난해한 코드로 인한 부채
- 테스트 부채: 테스트 부족이나 부실로 인한 부채

---

## 4. 기술 부채의 영향

- 유지 보수 비용 증가: 복잡한 코드로 인해 수정 및 유지 보수에 더 많은 시간과 비용 소요
- 버그 발생 가능성 증가: 품질이 낮은 코드로 인해 오류가 빈번히 발생
- 개발 속도 저하: 복잡한 코드 구조로 인해 기능 추가 및 수정이 어려움
- 팀 생산성 저하: 기술 부채 해결에 시간 소모 → 생산성 감소

---

## 5. 기술 부채 관리 방안

1. 주기적인 코드 리뷰: 정기적인 검토로 부채 조기 발견 및 해결
2. 자동화된 테스트 도입: 테스트 자동화로 코드 품질 유지
3. 지속적인 리팩토링: 지속적으로 코드를 개선해 부채 최소화
4. 명확한 문서화: 코드 및 시스템에 대한 문서화로 유지 보수 용이성 확보
- 기술 부채는 완전히 피할 수 없지만, 체계적인 관리와 예방을 통해 그 부정적인 영향을 최소화할 수 있음
