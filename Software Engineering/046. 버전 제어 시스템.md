# 버전 제어 시스템

---

## 1. Git과 SVN

- Git과 SVN(Subversion)은 소프트웨어 개발에서 널리 사용되는 버전 제어 시스템(VCS)이다.
- 각 프로젝트나 팀 상황에 따라 적합한 VCS를 선택하는 것이 중요하다.

---

## 2. Git과 SVN의 차이점

| 구분             | Git                                             | SVN                                      |
|------------------|-------------------------------------------------|------------------------------------------|
| 서버 아키텍처    | 분산형 (로컬에 저장소 복제, 서버와 클라이언트 역할 모두 수행) | 중앙 집중형 (서버와 클라이언트 구분)   |
| 개정 관리        | 전체 개정 번호 없음                             | 전체 개정 번호 있음                     |
| 저장소 복제      | 가능                                           | 불가능                                   |
| 저장 형식        | 메타데이터 중심                                 | 파일 중심                               |
| 저장소 용량      | 대용량 바이너리 파일 처리에 제약                | 대용량 바이너리 파일 처리 우수           |

## 3. SVN의 장점 (Git 대비)

- 우수한 아키텍처 성능
    - 네트워크 트래픽을 줄여주며, 팀과 개발자 간의 동기화가 빠르다.
    - Git의 경우 분산 시스템 특성상, 모든 변경 사항을 다운로드해야 하기 때문에 네트워크가 느린 경우 불편하다.
- 바이너리 파일 처리 우수
    - SVN은 Lock-Modify-Unlock 모델을 지원하여, 비병합 바이너리 파일 처리에 더 유리하다. 
    - Git은 분산 시스템 특성상 exclusive file locking을 지원하지 않아, 대용량 비병합 파일을 다룰 때 불편할 수 있다.
- 배우기 쉽고 사용 용이
    - SVN은 Git보다 구문이 간단하고 추상화가 잘 되어 있어 기능을 빠르게 익힐 수 있다.
- 강력한 접근 제어
    - SVN은 파일 수준 및 디렉토리 수준으로 세분화된 권한을 설정할 수 있어, 기업 환경에서 보안이 중요할 때 매우 유용하다. 
    - 반면 Git은 모든 사용자가 동일한 권한을 가지고 있기 때문에, 보안상 문제가 될 수 있다.

## 4. Git의 장점 (SVN 대비)

- 높은 인기와 생태계
    - Git은 GitHub와 같은 플랫폼을 통해 인기를 끌며, 오픈소스 개발과 협업의 중심으로 자리잡았다. 
    - 이는 Git의 선택을 이끌어낸 주요 요인 중 하나이다.
- 병합(Merge) 기능 우수
    - Git은 여러 개발자의 변경 사항을 병합하는 기능이 뛰어나다. 
    - 커뮤니티 협업을 중심으로 설계되어 병합 작업이 효율적으로 처리된다.
- 오프라인 작업 지원
    - Git의 분산형 아키텍처 덕분에 네트워크에 연결되지 않아도 로컬에서 작업을 이어나갈 수 있다.
    - 이후 온라인 상태에서 쉽게 푸시(push)할 수 있다.
- 네트워크 독립성
    - Git은 분산형 시스템으로 중앙 서버와 독립적으로 작업할 수 있다. 
    - 서버가 다운되거나 네트워크 연결이 끊겨도 작업을 계속할 수 있다는 점에서 유리하다.

---

## 5. 버전 관리 트렌드

- Git은 분산형 버전 관리 시스템으로, 오픈 소스 개발과 협업을 중심으로 최적화된 솔루션이다. 
  - GitHub와 같은 서비스 덕분에 더욱 널리 사용되고 있다.
- SVN은 여전히 기업 환경에서 선호되는 버전 관리 시스템으로, 특히 세밀한 권한 관리와 파일 기반 작업에 유리하다.
- 그 외의 버전 관리 시스템으로 Mercurial, Bazaar, CVS, Microsoft Azure DevOps도 존재한다.
