# 시퀸스 다이어그램

---

## 1. 개요

- 시퀀스 다이어그램은 특정 행동이 어떤 순서로 어떤 객체와 상호작용하는지를 표현하는 UML의 행위 다이어그램 
- 시스템의 시나리오를 시각적으로 파악할 수 있으며, API의 유즈 케이스나 타 시스템의 API 호출 로직을 모델링하는 데 유용

---

## 2. 주요 구성 요소

1. Lifeline
   - 모델링되는 인스턴스를 나타내며, 네모와 점선으로 구성 
   - 네모는 클래스나 컴포넌트를, 점선은 시간의 경과를 표현
2. Activation
   - Lifeline의 인스턴스가 다른 인스턴스와 상호작용하며 활성화된 상태를 나타내는 직사각형 막대

3. Message: 인스턴스 간에 주고받는 데이터를 표현하며, 다음과 같은 유형이 있습니다:
    - 동기 메시지(Synchronous Message): 요청을 보낸 후 결과가 올 때까지 기다리며, 실선과 꽉 찬 화살표로 표현
    - 비동기 메시지(Asynchronous Message): 요청을 보낸 후 결과를 기다리지 않고 다른 작업을 수행하며, 실선과 빈 화살표로 표현
    - 자체 메시지(Self Message): 자기 자신에게 요청을 보내는 경우로, 본인의 Lifeline으로 회귀하는 화살표로 표현
    - 반환 메시지(Return Message): 요청에 대한 결과를 반환하며, 점선과 빈 화살표로 표현

---

## 3. 흐름 제어 요소:

- Guard: 
  - 단일 메시지에 대한 조건을 명시하며, 메시지 앞에 대괄호([])로 감싸서 표현합니다. 
  - 예: [price > 10000]
- Sequence Fragment: 특정 부분에 대한 흐름 제어를 위해 사용되며, 주요 유형은 다음과 같습니다:
    - Alternative (alt): 조건에 따라 여러 선택 사항이 있을 때 사용하며, if/else 문과 유사
    - Option (opt): 단일 조건에 따른 선택 사항이 있을 때 사용하며, if 문과 유사
    - Loop: 특정 조건이 만족될 때까지 반복 실행되며, for 또는 while 문과 유사
    - Parallel (par): 병렬 처리를 표현하며, 동시에 수행되는 작업을 나타냄
