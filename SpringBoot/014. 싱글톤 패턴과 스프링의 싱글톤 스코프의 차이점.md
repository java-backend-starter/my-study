# 싱글톤 패턴과 스프링의 싱글톤 스코프의 차이점

---

## 1. 싱글톤 패턴이란?

- 싱글톤 패턴은 특정 클래스의 인스턴스가 하나만 생성되도록 보장하는 디자인 패턴이다. 
- 메모리 낭비를 방지하고, 공통된 자원(예: DB 연결 풀)을 효율적으로 관리할 수 있다.

---

## 자바 싱글톤 패턴

### 구현 방법

1. 생성자 private 설정: 외부에서 인스턴스를 생성할 수 없도록 한다.
2. static 변수로 인스턴스 선언: 클래스 로딩 시 한 번만 메모리에 로드되도록 한다.
3. getInstance() 메소드 제공: 인스턴스를 반환하는 메소드를 제공한다.
- Thread Safety를 보장하기 위해 `getInstance()` 메소드에 `synchronized`를 추가하거나, `Bill Pugh` 방식을 사용할 수 있다.

---

## 스프링 싱글톤 패턴

- 스프링에서는 싱글톤 패턴을 다음과 같이 구현:
  - 스프링 컨테이너 관리: 스프링 컨테이너가 객체의 생명주기를 관리하며, 기본적으로 빈(Bean)의 범위는 싱글톤입니다.
  - Thread Safety 보장: 스프링은 내부적으로 싱글톤 빈의 스레드 안전성을 보장합니다.

---

## 주요 차이점 요약

| 구분             | 자바 싱글톤 패턴                          | 스프링 싱글톤 패턴                         |
|------------------|------------------------------------------|-------------------------------------------|
| 관리 주체        | 개발자                                    | 스프링 컨테이너                           |
| 인스턴스 생성 시점 | 클래스 로딩 시점                          | 컨테이너 초기화 시점 또는 최초 호출 시점  |
| Thread Safety    | 개발자 구현에 따라 다름                   | 자동으로 보장됨                           |
| 범위(Scope)      | 애플리케이션 전체                         | 스프링 컨테이너 내에서만 유효             |
