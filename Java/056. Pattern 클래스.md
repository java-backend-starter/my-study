# Pattern 클래스

---

## 1. 개요

- 문자열이 정해진 형식(정규 표현식: Regular Expression)으로 구성되어 있는지 검증해야 할 때가 있다.
- 예를 들어, 이메일이나 전화번호가 사용자가 제대로 입력했는지 검증하는 경우가 이에 해당한다.

---

## 2. 정규 표현식 작성 방법

- 정규 표현식을 작성하는 방법은 java.util.regex.Pattern 클래스의 API 도큐먼트에서 "Summary of regular-expression constructs"를 참조하면 되지만, 이해하기 어려울 수 있다. 
- 간단히 말해 정규 표현식은 문자 또는 숫자 기호와 반복 기호가 결합된 문자열이다. 
- 아래는 정규 표현식을 작성할 때 기본적으로 알아야 할 기호들에 대한 설명이다.


### 기호	설명

| 기호            | 설명                                                    |
|-----------------|---------------------------------------------------------|
| `[ ]`           | 한 개의 문자                                           |
| `[^(기호)]`      | 기호 이외의 하나의 문자                                 |
| `[기호-기호]`    | 해당 범위의 기호들 중 하나의 문자                       |
| `\d`            | 한 개의 숫자(`=[0-9]`)                                 |
| `\s`            | 공백                                                    |
| `\w`            | 한 개의 알파벳 또는 한 개의 숫자(`=[a-z_A-Z_0-9]`)        |
| `?`             | 없음 또는 한 개                                         |
| `*`             | 없음 또는 한 개 이상                                    |
| `+`             | 한 개 이상                                              |
| `{n}`           | 정확히 n개                                              |
| `{n, }`         | 최소한 n개                                              |
| `{n, m}`        | n개에서 m개까지                                         |
| `( )`           | 그룹핑                                                  |

     
### 예시: 전화번호 정규 표현식
    
-  전화번호 예시: 02-123-1234 또는 010-1234-5678과 같은 전화번호를 위한 정규 표현식은 다음과 같다.
```java
(02|010)-\d{3,4}-\d{4}
```

### 예시: 이메일 정규 표현식

- 이메일 예시: white@naver.com과 같은 이메일을 위한 정규 표현식은 다음과 같다.
```java
\w+@\w+\.\w+(\.\w+)?
```
- 주의할 점은 \.과 .은 다르다는 것이다. 
- \.은 문자로서의 점 (.)을 의미하고, .은 모든 문자 중 한 개의 문자를 의미한다.

---

## 3. Pattern 클래스

- 문자열을 정규 표현식으로 검증하는 기능은 java.util.regex.Pattern 클래스의 정적 메소드인 matches() 메소드에서 제공된다.
```java
boolean result = Pattern.matches("정규식", "검증할 문자열");
```
- 첫 번째 매개값은 정규 표현식이고, 두 번째 매개값은 검증할 문자열이다. 
- 검증 후 결과는 boolean 타입으로 반환된다.

### 예시: 전화번호 및 이메일 검증

```java
// 문자열 검증하기
public class PatternExample {
    public static void main(String[] args) {
        // 전화번호 정규식
        String regExp = "(02|010)-\\d{3,4}-\\d{4}";
        String data = "010-123-4567";
        boolean result = Pattern.matches(regExp, data);

        if (result) {
            System.out.println("정규식과 일치합니다.");
        } else {
            System.out.println("정규식과 일치하지 않습니다.");
        }

        // 이메일 정규식
        regExp = "\\w+@\\w+\\.\\w+(\\.\\w+)?";
        data = "angel@navercom";
        result = Pattern.matches(regExp, data);
        
        if (result) {
            System.out.println("정규식과 일치합니다.");
        } else {
            System.out.println("정규식과 일치하지 않습니다.");
        }
    }
}
```
- 이 예제에서 전화번호 "010-123-4567"은 정규식과 일치하므로 "정규식과 일치합니다."가 출력된다. 
- 이메일 "angel@navercom"은 정규식과 일치하지 않기 때문에 "정규식과 일치하지 않습니다."가 출력된다. 
- 이메일이 "angel@naver.com"처럼 점 (.)이 포함되어야 정상적인 이메일 형식이 된다. 
- 따라서 정규 표현식은 유효한 문자열을 검증하는 데 매우 유용한 도구입니다.
