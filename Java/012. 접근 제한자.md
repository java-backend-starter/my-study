# 접근 제한다(Access Modifier)

---

## 1. 접근 제한자 개요

* 라이브러리 클래스: 외부에서 사용하도록 설계됨
* 접근 제한 필요
  * 특정 멤버만 접근 허용
  * 객체 생성을 막거나 데이터 보호
  * 특정 메소드 호출 제한  
* 자바의 해결책: 접근 제한자(Access Modifier) 제공
* 접근 제한자의 종류
  * public: 모든 클래스에서 접근 가능 
  * protected: 같은 패키지 + 자식 클래스에서 접근 가능 
  * default (명시 없음): 같은 패키지에서만 접근 가능 
  * private: 해당 클래스 내부에서만 접근 가능

## 2. 클래스의 접근 제한

* 클래스 선언 시 고려 사항
  * 같은 패키지에서만 사용할지, 다른 패키지에서도 사용할지 결정
* 클래스의 접근 제한
  * public 클래스: 모든 패키지에서 사용 가능
  * default 클래스: 같은 패키지에서만 사용 가능

## 3. 생성자의 접근 제한

* 객체 생성: new 연산자로 생성자 호출
* 제한 사항: 생성자의 접근 제한자에 따라 호출 가능 여부 결정
* 생성자의 접근 제한
  * public 생성자: 모든 패키지에서 객체 생성 가능
  * protected 생성자: 같은 패키지 + 자식 클래스에서 객체 생성 가능
  * default 생성자: 같은 패키지에서만 객체 생성 가능
  * private 생성자: 외부에서 객체 생성 불가 → 싱글톤 패턴에 사용

## 4. 필드와 메소드의 접근 제한

* 필드와 메소드 선언 시 고려 사항
  * 클래스 내부에서만 사용 
  * 패키지 내에서만 사용 
  * 다른 패키지에서도 사용
* 접근 제한자에 따라 사용 범위 결정
  * public: 모든 패키지에서 접근 가능 
  * protected: 같은 패키지 + 자식 클래스에서 접근 가능 
  * default: 같은 패키지에서만 접근 가능 
  * private: 해당 클래스 내부에서만 접근 가능
