# Java Record

---

## 1. record

- Record는 변경이 불가능한 데이터 객체를 쉽게 만들 수 있도록 제공되는 Java 문법이다.
- 기존 클래스보다 간결하고 효율적으로 데이터 객체를 표현할 수 있도록 설계되었다.
- Java 14: Preview 기능
- Java 16: 정식 기능

---

## 2. 목표

- 데이터를 간결하게 표현
- 개발자가 불변 데이터 모델링에 집중하도록 지원
- 데이터 중심 메서드를 자동 구현

---

## 3. 특징

1. 간결성
    - 필드 정의만으로 반복 코드 자동 생성
    - 보일러플레이트 코드 제거
2. 메서드 자동 생성
    - equals()
    - hashCode()
    - toString()
3. 생성자 자동 생성
    - 모든 컴포넌트를 받는 생성자 자동 생성
    - 기본 생성자는 제공되지 않음
4. 불변성
    - 필드 값 변경 불가
    - setter 미제공
5. final 선언 생략
    - 모든 필드는 암시적으로 private final
    - abstract 선언 불가
6. 데이터 전달에 적합
    - DTO(Data Transfer Object)에 최적화

### 보일러플레이트 코드란?

- 여러 곳에서 반복적으로 사용되는 기본 코드
    - getter / setter
    - equals()
    - hashCode()
    - toString()

---

## 4. 제약사항

- abstract일 수 없어 다른 클래스를 상속할 수 없음
- 인스턴스 필드는 private final만 가능
- static 필드 / 메서드는 선언 가능
- 한 번 값이 정해지면 setter를 통해 값을 변경할 수 없음 (불변 객체)

---

## 5. 클래스와 동일하게 가능한 것

- new 키워드로 객체 생성
- static 필드 / 메서드 선언
- 중첩 클래스 사용
- 제네릭 타입 사용
- 인터페이스 구현 가능

---

## 6. Class vs Record

### Class (AS-IS)

```java
public class User {
    private Long id;
    private String name;
    private int age;

    // 생성자, getter/setter, equals, hashCode, toString ...
}
```

### Record (TO-BE)

```java
public record User(Long id, String name, int age) { }

User user = new User(1L, "김개발", 25);

System.out.println(user.id());
System.out.println(user.name());
System.out.println(user.age());
```

### Record 구조 설명

- 헤더(Header): 컴포넌트 정의
- 바디(Body): 추가 로직 작성 가능
- 헤더에 정의된 필드를 컴포넌트(Component) 라고 함
- 컴파일러가 자동 생성:
    - 필드
    - 접근자 메서드
    - 생성자
    - toString(), equals(), hashCode()

### 컴팩트 생성자 (Compact Constructor)

#### 컴팩트 생성자란?

- Record의 생성자가 단순 초기화 외에 추가 작업을 해야 할 때 사용하는 생성자이다.
- 주로 유효성 검사(validation) 용도로 사용된다.
- 일반 클래스 생성자와 달리 매개변수 목록이 없다.

#### 컴팩트 생성자 예시

```java
import java.util.Objects;

public record Person(String name, String address) {
    public Person {
        Objects.requireNonNull(name);
        Objects.requireNonNull(address);
    }
}
```

- 컴파일러가 자동으로 필드 초기화
- 개발자는 검증 로직만 작성

#### 다른 인자를 받는 생성자 추가 가능

```java
public record Person(String name, String address) {
    public Person(String name) {
        this(name, "Unknown");
    }
}
```

####  명시적 생성자 작성 시 주의

- 컴포넌트와 동일한 매개변수 목록의 생성자를 만들면 필드를 직접 초기화해야 한다
```java
public record Person(String name, String address) {
    public Person(String name, String address) {
        this.name = name;
        this.address = address;
    }
}
```

#### 컴파일 오류 발생 사례

- 컴팩트 생성자와 동일한 시그니처의 생성자를 동시에 선언할 수 없다
```java
public record Person(String name, String address) {
    public Person {
        Objects.requireNonNull(name);
    }

    // ❌ 컴파일 에러
    public Person(String name, String address) {
        this.name = name;
        this.address = address;
    }
}
```
