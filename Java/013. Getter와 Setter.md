# Getter와 Setter

---

## 1. 개요

* 멤버 변수를 public으로 선언하면 외부에서 직접 접근 가능하지만, 이는 예기치 않은 동작을 초래할 수 있음 
* private 선언 후, 외부에서 접근할 방법이 필요

## 2. Getter/Setter

* private 멤버 변수에 접근할 수 있도록 해주는 메서드 
* 클래스 내부 구현 변경 시, 외부 코드에 영향 최소화

## 3. 장점

* 캡슐화 & 정보 은닉 유지 
* 필드 값 제어 가능 (유효성 검사 가능)

## 4. 단점

* 코드 복잡성 증가 (과도한 사용 시 가독성 저하)
* 성능 오버헤드 발생 (메서드 호출 추가)
