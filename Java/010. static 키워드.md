# static 키워드 

---

## 1. 정적 (static) 멤버

* static은 "고정된" 이라는 의미를 가지며, 클래스에 고정된 멤버를 의미한다.
* 객체를 생성하지 않고 사용할 수 있는 필드와 메서드를 정적 멤버라고 한다.
* 정적 멤버는 클래스에 소속되므로 클래스 멤버라고도 한다.

## 2. 정적 멤버 선언

* 정적 필드 : static 키워드를 사용하여 선언하며, 모든 객체가 공유하는 변수이다.
* 정적 메서드 : static 키워드를 사용하여 선언하며, 객체 없이 호출 가능하다.
* 선언 방법
```java
public class 클래스 {
    // 정적 필드
    static 타입 필드 [= 초깃값];
    
    // 정적 메소드
    static 리턴타입 메소드(매개변수 선언, ...){
        // ...
    }
}
```
* 정적 멤버는 클래스 로더가 메모리에 적재할 때 생성되므로, 클래스 로딩 후 바로 사용 가능하다.
* 인스턴스 필드 vs. 정적 필드 판단 기준 
  * 객체마다 다른 값을 가져야 하면 → 인스턴스 필드 
  * 모든 객체가 공통으로 사용하는 값이면 → 정적 필드
* 인스턴스 메서드 vs. 정적 메서드 판단 기준 
  * 인스턴스 필드를 활용하면 → 인스턴스 메서드 
  * 인스턴스 필드를 사용하지 않으면 → 정적 메서드

## 3. 정적 멤버 사용법

* 클래스 이름을 사용하여 호출 
  * 클래스명.정적필드 
  * 클래스명.정적메서드(매개변수)
* 객체를 통해 접근하는 것도 가능하지만 권장되지 않음
* 객체를 통해 정적 멤버에 접근하면 경고 메시지가 표시될 수 있음

## 4. 정적 초기화 블록 (static 블록)

* 정적 필드의 복잡한 초기화를 수행할 때 사용 
* 클래스가 메모리에 로딩될 때 자동 실행 
* 여러 개 선언 가능하며, 선언 순서대로 실행됨 
* 생성자에서 정적 필드를 초기화할 수 없으므로 static 블록을 사용
* 형태
```java
static {
    // 초기화 내용
}
```
* 클래스 로딩 시 실행되므로 한 번만 실행됨

## 5. 정적 메서드와 블록 선언 시 주의할 점

* 객체가 없어도 실행되므로, 인스턴스 필드나 인스턴스 메서드를 사용할 수 없음 
* this 키워드 사용 불가 
* 정적 메서드나 블록에서 인스턴스 멤버를 사용하려면 객체를 생성해야 함 
* main() 메서드도 정적 메서드이므로 인스턴스 멤버를 직접 사용할 수 없음
