# 객체 지향 프로그래밍

---

## 1. 객체 지향 프로그래밍(OOP) 개념

* 현실 세계에서 부품을 조립해 제품을 만들듯, 객체(부품)를 조립하여 프로그램을 개발하는 방식
* 독립적인 객체들을 설계하고 조합해 소프트웨어를 구성.

## 2. 객체(Object)란?

* 물리적(자동차, 사람) 또는 추상적(학과, 주문)으로 존재하며, 속성과 동작을 가짐. 
* 자바에서는 속성을 필드(Field), 동작을 메소드(Method) 라고 부름. 
* 현실 세계 객체를 소프트웨어로 설계하는 과정을 객체 모델링(Object Modeling) 이라 함.

## 3. 객체 간의 관계

* 집합 관계: 한 객체가 다른 객체를 포함하는 관계 (예: 자동차 → 엔진, 타이어). 
* 사용 관계: 한 객체가 다른 객체의 기능을 이용하는 관계 (예: 사람이 자동차를 운전). 
* 상속 관계: 부모 객체의 속성과 동작을 자식 객체가 물려받음 (예: 기계 → 자동차).

## 4. 객체 간 상호작용

* 객체들은 독립적으로 존재하며, 메소드 호출을 통해 상호작용.
* 예: 계산기.add(10, 20); → add() 메소드가 실행되어 결과 반환.

## 5. 객체 지향 프로그래밍(OOP)의 주요 특징

* 캡슐화(Encapsulation)
  * 객체의 필드와 메소드를 하나로 묶고, 외부에서 직접 접근하지 못하도록 보호. 
  * 접근 제한자(Access Modifier)를 사용하여 정보 은닉.
  * 예: TV 내부 부품을 보호하여 잘못된 조작 방지.
* 상속(Inheritance)
  * 기존 객체(부모)의 필드와 메소드를 새로운 객체(자식)가 재사용하도록 함.
  * 코드 중복을 줄이고 유지보수를 쉽게 만듦.
  * 예: 기계 클래스가 자동차, 로봇 등의 하위 클래스에 공통 기능을 제공.
* 다형성(Polymorphism)
  * 동일한 인터페이스나 부모 클래스를 기반으로 다양한 객체를 사용할 수 있음.
  * 부모 타입 변수로 다양한 자식 객체를 참조할 수 있음.
  * 예: 타이어 인터페이스를 구현한 여러 회사의 타이어가 자동차에 장착 가능.
